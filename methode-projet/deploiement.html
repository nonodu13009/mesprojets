Déploiement

Étape 6 · Mettre l'app en ligne avec Vercel

Le grand moment

Ton app fonctionne en local. Elle est testée. Il est temps de la rendre accessible à tes utilisateurs. Avec Vercel et Claude Code, le déploiement est devenu une opération de quelques minutes — pas de serveur à configurer, pas de DevOps.

Étape 1 : Préparer le déploiement

Avant de déployer, vérifie ces points :

Vérification Commande / Action
Le code compile sans erreur npm run build
Pas de secrets dans le code Vérifie qu'aucune clé API n'est en dur
Les variables d'environnement sont dans .env.local Vérifie le fichier .env.example
Le .gitignore exclut .env* Vérifie le fichier
Tout est commité et pushé /commit

Étape 2 : Déployer avec Claude Code

Claude Code intègre Vercel nativement. Le process en 3 commandes : 

// 1. Configurer Vercel (première fois seulement)
"/vercel:setup"

// 2. Déployer
"/vercel:deploy"

// 3. Vérifier les logs si problème
"/vercel:logs"

Vercel va :

  1. Détecter automatiquement que c'est un projet Next.js
  2. Builder l'app
  3. La déployer sur une URL temporaire (ton-projet.vercel.app)
  4. Te donner le lien

Étape 3 : Variables d'environnement

Les clés API (Supabase, Stripe, etc.) ne doivent jamais être dans le code. Configure-les dans le dashboard Vercel : 

// Dans le dashboard Vercel :
// Settings → Environment Variables

NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbG...
SUPABASE_SERVICE_ROLE_KEY=eyJhbG...

Important : les variables qui commencent par NEXT_PUBLIC_ sont visibles côté client (navigateur). Ne mets jamais de clé secrète avec ce préfixe.

Étape 4 : Domaine personnalisé

L'URL ton-projet.vercel.app fonctionne, mais pour un vrai projet, tu veux un domaine propre :

  1. Achète un domaine (Vercel, Namecheap, OVH — ~10€/an)
  2. Dans Vercel → Settings → Domains → ajoute ton domaine
  3. Configure les DNS selon les instructions de Vercel
  4. Le SSL (HTTPS) est automatique

Étape 5 : Déploiement automatique

Une fois configuré, Vercel redéploie automatiquement à chaque git push sur la branche main. Ton workflow devient :

CodeClaude Code → Commit/commit → PushGitHub → LiveVercel (auto)

Tu ne toucheras plus jamais au déploiement manuellement. Chaque /commit met à jour l'app en production en quelques secondes.

Le livrable

  1. L'app est en ligne et accessible
  2. Les variables d'environnement sont configurées
  3. Le domaine est connecté (optionnel pour le MVP)
  4. Le déploiement automatique fonctionne

La suite

Ton app est en production. Des vrais utilisateurs vont l'utiliser. Et c'est là que le vrai travail commence : écouter le feedback, corriger les bugs de production, prioriser les améliorations et planifier l'évolution. C'est l'étape Après.
