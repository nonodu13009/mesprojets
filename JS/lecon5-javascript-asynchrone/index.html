Leçon 5

Le JavaScript Asynchrone

Où en es-tu ?

Jusqu'ici, ton code s'exécute ligne après ligne, dans l'ordre, instantanément. Mais le web réel est plein d'opérations qui prennent du temps : appeler une API, charger une image, attendre un minuteur, lire un fichier...

Si JavaScript attendait la fin de chaque opération avant de continuer, la page serait gelée pendant ce temps : plus de clics, plus de scroll, plus rien. La solution : l'asynchrone — lancer une opération, continuer à travailler, et réagir quand le résultat arrive.

C'est un cap conceptuel important : cette leçon le franchit en trois étapes, dans l'ordre historique du langage — chaque outil corrige les défauts du précédent.

Ce que tu vas apprendre

Cette leçon est organisée en 3 chapitres progressifs, avec des exemples concrets à tester dans la console.

Comment aborder cette leçon ?

Suis les chapitres dans l'ordre, c'est essentiel ici : les promesses n'ont de sens que si tu as compris les limites des callbacks, et async/await n'est qu'une écriture plus confortable... des promesses. C'est une seule et même histoire, racontée en trois chapitres.

Les exemples utilisent setTimeout (un minuteur) pour simuler la lenteur, puis fetch pour appeler de vraies API. Tout se teste dans la console, comme d'habitude.

Prérequis

Avant de commencer, assure-toi d'être à l'aise avec les fonctions fléchées et le fait de passer une fonction en argument d'une autre (Leçon 3, chapitre 24) : tout l'asynchrone repose là-dessus.
