Introduction au DOM

Leçon 4 · Chapitre 30 · Manipule le document web dynamique

C'est quoi, le DOM ?

Quand le navigateur charge une page HTML, il ne se contente pas de l'afficher : il la transforme en une structure d'objets JavaScript appelée le DOM (Document Object Model).

Chaque balise HTML devient un objet (un nœud) que ton code peut lire et modifier. Le HTML est le plan de départ ; le DOM est la version vivante de la page, en mémoire. 

<!-- Le HTML que tu écris... -->
<body>
    <h1>Bienvenue</h1>
    <p>Bonjour à tous</p>
</body>

// ...devient un arbre d'objets accessibles en JavaScript :
//   document
//   └── body
//       ├── h1  → "Bienvenue"
//       └── p   → "Bonjour à tous"

C'est pour ça qu'on parle d'arbre DOM : chaque élément a un parent, et éventuellement des enfants — exactement comme l'imbrication des balises HTML.

La porte d'entrée : l'objet document

Tout part de l'objet global document, qui représente la page entière. Souviens-toi de la leçon 3 : un objet a des propriétés et des méthodes. document n'échappe pas à la règle. 

// À tester dans la console, sur n'importe quelle page !
console.log(document.title);           // le titre de l'onglet
console.log(document.URL);             // l'adresse de la page
console.log(document.body);            // l'élément <body> complet
console.log(document.body.children);   // ses enfants directs

Astuce : dans la console, tape document puis explore l'objet en dépliant les flèches. Tu verras toute la structure de la page.

Première manipulation : modifier la page

La magie du DOM : si tu modifies un objet, la page se met à jour instantanément. Essaie ça dans la console, ici même : 

// Change le titre de l'onglet
document.title = "Mon nouveau titre";

// Change la couleur de fond de toute la page
document.body.style.backgroundColor = "lightyellow";

// Remplace le premier titre de la page
document.querySelector("h1").textContent = "Le DOM, c'est moi !";

Recharge la page (F5) et tout revient à la normale : tu as modifié le DOM (la version en mémoire), pas le fichier HTML.

Les nœuds de l'arbre

Chaque élément du DOM connaît sa position dans l'arbre. Tu peux naviguer de parent en enfant : 

const titre = document.querySelector("h1");

console.log(titre.parentElement);          // l'élément qui le contient
console.log(titre.nextElementSibling);     // l'élément juste après
console.log(document.body.firstElementChild); // le premier enfant du body

Tu utiliseras rarement cette navigation manuelle (les sélecteurs du prochain chapitre sont plus pratiques), mais elle t'aide à comprendre la logique d'arbre : parent, enfants, frères et sœurs.

Pourquoi c'est fondamental ?

Tout ce que tu vois bouger sur un site — un menu qui s'ouvre, un compteur qui s'incrémente, un formulaire qui affiche une erreur, un panier qui se met à jour — c'est du JavaScript qui manipule le DOM.

Même les frameworks comme React ou Vue ne font "que" ça : ils manipulent le DOM à ta place, de façon optimisée. Comprendre le DOM, c'est comprendre comment le web fonctionne vraiment.

À toi de jouer

  1. Ouvre la console (F12 → Console) sur cette page
  2. Affiche document.title puis change-le
  3. Tape document.body et survole le résultat : le navigateur surligne l'élément dans la page
  4. Change la couleur de fond avec document.body.style.backgroundColor = "lavender"
  5. Remplace le texte du h1 avec querySelector et textContent
  6. Recharge la page et constate que tout revient à la normale
