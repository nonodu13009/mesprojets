Manipuler le CSS avec le DOM

Leçon 4 · Chapitre 32 · Styles dynamiques et classes

Deux approches pour styler en JavaScript

Pour changer l'apparence d'un élément depuis JavaScript, tu as deux outils :

  • La propriété style : modifier un style directement, propriété par propriété.
  • classList : ajouter ou retirer des classes CSS définies dans ta feuille de style.

Les deux sont utiles, mais tu vas voir que la deuxième est presque toujours la meilleure.

1. La propriété style

Chaque élément du DOM a une propriété style qui reflète ses styles inline (l'attribut style="" du HTML) : 

const titre = document.querySelector("h1");

titre.style.color = "crimson";
titre.style.fontSize = "3rem";
titre.style.backgroundColor = "lightyellow";
titre.style.padding = "1rem";

Règle de conversion : les propriétés CSS à tiret passent en camelCase en JavaScript. font-size → fontSize, background-color → backgroundColor, border-radius → borderRadius. Et les valeurs sont toujours des chaînes, unité comprise : "3rem", pas 3.

Pour lire les styles réellement appliqués (y compris ceux venant des feuilles CSS), element.style ne suffit pas — il ne voit que l'inline. Utilise getComputedStyle : 

const stylesReels = getComputedStyle(titre);
console.log(stylesReels.fontSize);  // ex. "40px"
console.log(stylesReels.color);     // ex. "rgb(29, 29, 31)"

2. classList : la méthode propre

Modifier 6 propriétés style une par une, c'est verbeux et ça mélange le design (CSS) et la logique (JS). La bonne pratique : définir une classe CSS et la brancher/débrancher depuis JavaScript. 

/* Dans ta feuille CSS */
/* .surbrillance {
    background-color: gold;
    border-radius: 8px;
    padding: 0.5rem;
} */

const titre = document.querySelector("h1");

titre.classList.add("surbrillance");      // ajoute la classe
titre.classList.remove("surbrillance");   // la retire
titre.classList.toggle("surbrillance");   // ajoute si absente, retire si présente
console.log(titre.classList.contains("surbrillance")); // true / false

toggle est parfait pour tout ce qui s'ouvre et se ferme : menu burger, mode sombre, accordéon, modale...

style ou classList : que choisir ?

Situation Outil
État visuel prédéfini (actif, caché, erreur, mode sombre) classList ✅
Valeur calculée en JS (position d'un élément qu'on déplace, largeur d'une barre de progression) style
Tout le reste classList par défaut

En résumé : le CSS décrit les apparences possibles, le JavaScript décide laquelle s'applique et quand. Chacun son métier.

Exemple concret : une barre de progression

Un cas où les deux approches se combinent naturellement : 

// La classe CSS définit l'apparence générale (couleur, hauteur, transition)
// Le JS calcule la largeur, impossible à prévoir en CSS

const barre = document.createElement("div");
barre.classList.add("barre-progression");

const chapitresLus = 3;
const chapitresTotal = 32;
const pourcentage = (chapitresLus / chapitresTotal) * 100;

barre.style.width = `${pourcentage}%`; // valeur calculée → style

document.body.prepend(barre);

Cacher / afficher un élément

Le besoin le plus fréquent du web dynamique. Trois options à connaître : 

const encart = document.querySelector(".note");

// Option 1 : display none (l'élément disparaît et libère sa place)
encart.style.display = "none";
encart.style.display = ""; // retour au style de la feuille CSS

// Option 2 : l'attribut hidden (équivalent, plus sémantique)
encart.hidden = true;
encart.hidden = false;

// Option 3 : une classe .cache { display: none; } + toggle
encart.classList.toggle("cache");

À toi de jouer

  1. Ouvre la console (F12 → Console) sur cette page
  2. Sélectionne le h1 et change sa couleur avec style.color
  3. Teste la conversion camelCase : style.backgroundColor, style.borderRadius
  4. Lis la taille de police réelle avec getComputedStyle
  5. Ajoute une classe existante du site (ex. note) à un paragraphe avec classList.add
  6. Fais disparaître puis réapparaître un élément avec hidden
  7. Recrée la barre de progression avec tes propres valeurs
