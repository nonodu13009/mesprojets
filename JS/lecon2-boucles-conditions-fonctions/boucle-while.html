La boucle while et do...while

Leçon 2 · Chapitre 8 · Répéter tant qu'une condition est vraie

Quand for ne suffit pas

Au chapitre précédent, tu as appris la boucle for — idéale quand tu connais le nombre de répétitions à l'avance. Mais dans beaucoup de situations, tu ne sais pas combien de tours seront nécessaires :

  • Redemander un mot de passe jusqu'à ce qu'il soit correct
  • Lancer un dé jusqu'à obtenir un 6
  • Lire des données jusqu'à la fin d'un fichier

Pour ces cas, JavaScript propose la boucle while : elle répète un bloc tant que sa condition reste true.

La boucle while

let compteur = 0;

while (compteur < 5) {
    console.log(`Tour n°${compteur}`);
    compteur++;
}
// Tour n°0
// Tour n°1
// Tour n°2
// Tour n°3
// Tour n°4

Le principe est simple : avant chaque tour, JavaScript vérifie la condition. Si elle est true, le bloc s'exécute. Si elle est false, la boucle s'arrête et le programme continue après.

Attention : si la condition ne devient jamais false, la boucle tourne à l'infini et bloque le navigateur ! Pense toujours à modifier la variable testée dans la condition.

while vs for : quand utiliser quoi ?

Boucle Quand l'utiliser Exemple typique
for Nombre de tours connu à l'avance Parcourir un tableau, compter de 1 à 10
while Nombre de tours inconnu Attendre une saisie correcte, chercher une valeur

Exemple : deviner un nombre

const secret = 7;
let tentative = 0;

while (tentative !== secret) {
    tentative = Number(prompt("Devine le nombre (1-10) :"));
}

console.log("Bravo, tu as trouvé !");

Ici, impossible de savoir combien de tentatives l'utilisateur aura besoin. La boucle while est donc le choix parfait : on répète tant que la réponse est fausse.

La boucle do...while

La variante do...while a une différence fondamentale : le bloc s'exécute au moins une fois, car la condition est testée après l'exécution.

whileteste AVANT → peut ne jamais s'exécuter do...whileteste APRÈS → s'exécute au moins 1 fois 
let réponse;

do {
    réponse = prompt("Tape 'oui' pour continuer :");
} while (réponse !== "oui");

console.log("Merci !");

Le prompt s'affiche au moins une fois, puis se répète tant que l'utilisateur ne tape pas « oui ». C'est logique : on doit poser la question avant de pouvoir vérifier la réponse.

Exemple : lancer un dé jusqu'à obtenir un 6

let dé;
let nbLancers = 0;

do {
    dé = Math.ceil(Math.random() * 6);
    nbLancers++;
    console.log(`Lancer ${nbLancers} : ${dé}`);
} while (dé !== 6);

console.log(`Tu as obtenu un 6 en ${nbLancers} lancers !`);

Math.random() génère un nombre aléatoire entre 0 et 1. Math.ceil(Math.random() * 6) donne un entier entre 1 et 6. Tu découvriras d'autres méthodes de Math au fil des leçons.

Éviter les boucles infinies

C'est le piège classique du débutant. Si tu oublies de modifier la variable testée dans la condition, la boucle ne s'arrêtera jamais : 

// DANGER : boucle infinie !
let x = 0;
while (x < 10) {
    console.log(x);
    // On a oublié x++ → x reste à 0 pour toujours
}

// CORRECT :
let y = 0;
while (y < 10) {
    console.log(y);
    y++; // Ne pas oublier !
}

Astuce : si ton navigateur se fige à cause d'une boucle infinie, ferme l'onglet (pas tout le navigateur). Sur Chrome, tu peux aussi ouvrir le Gestionnaire de tâches (⇧ + Esc) pour tuer l'onglet.

À toi de jouer

  1. Ouvre la console (⌘ + ⌥ + J)
  2. Teste une boucle while simple avec un compteur
  3. Essaie le lancer de dé avec do...while
  4. Compare : que se passe-t-il avec while (false) { ... } vs do { ... } while (false) ?

La suite

Tu maîtrises maintenant les deux types de boucles : for pour un nombre connu de répétitions, while pour un nombre inconnu. Mais une boucle seule ne suffit pas — il faut pouvoir prendre des décisions : exécuter un bloc de code si une condition est remplie, un autre bloc sinon. C'est exactement ce que font les structures if et else.
