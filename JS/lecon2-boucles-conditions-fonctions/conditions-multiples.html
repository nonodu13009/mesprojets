Les conditions multiples

Leçon 2 · Chapitre 10 · Gérer plusieurs cas avec if...else if...else

Au-delà du vrai/faux

Au chapitre précédent, tu as appris if...else — un choix entre deux options. Mais la réalité est rarement binaire. Pour attribuer une mention à un examen, calculer un tarif selon l'âge, ou afficher un message selon le jour, il faut pouvoir tester plusieurs conditions les unes après les autres.

C'est le rôle de la structure if...else if...else : elle teste les conditions dans l'ordre, et exécute le premier bloc dont la condition est vraie.

Syntaxe if...else if...else

const note = 15;

if (note >= 16) {
    console.log("Très bien");
} else if (note >= 14) {
    console.log("Bien");
} else if (note >= 12) {
    console.log("Assez bien");
} else if (note >= 10) {
    console.log("Passable");
} else {
    console.log("Insuffisant");
}
// "Bien" (15 >= 14)

Important : les conditions sont testées dans l'ordre, de haut en bas. Dès que l'une est true, son bloc s'exécute et toutes les suivantes sont ignorées — même si elles seraient aussi vraies.

L'ordre des conditions compte !

C'est un piège classique. Si tu mets les conditions dans le mauvais ordre, tu obtiens des résultats incorrects : 

const note = 18;

// MAUVAIS ordre :
if (note >= 10) {
    console.log("Passable"); // 18 >= 10 → true, on entre ici !
} else if (note >= 16) {
    console.log("Très bien"); // Jamais atteint pour 18
}

// BON ordre (du plus restrictif au moins restrictif) :
if (note >= 16) {
    console.log("Très bien"); // 18 >= 16 → true, correct !
} else if (note >= 10) {
    console.log("Passable");
}

Règle : quand tu compares des valeurs numériques avec des intervalles, commence toujours par la condition la plus restrictive (la plus grande valeur) et descends progressivement.

Exemple : catégorie d'âge

const age = 25;

if (age < 3) {
    console.log("Bébé");
} else if (age < 12) {
    console.log("Enfant");
} else if (age < 18) {
    console.log("Adolescent");
} else if (age < 65) {
    console.log("Adulte");
} else {
    console.log("Senior");
}
// "Adulte"

Ici, l'ordre est naturel (du plus petit au plus grand) car on utilise < au lieu de >=. Les deux approches fonctionnent, tant que la logique reste cohérente.

Exemple : tarif cinéma

Un cas réaliste qui combine âge, statut et opérateurs logiques : 

const age = 14;
const estÉtudiant = true;
let tarif;

if (age < 4) {
    tarif = 0;           // Gratuit pour les moins de 4 ans
} else if (age < 12) {
    tarif = 6;           // Tarif enfant
} else if (estÉtudiant) {
    tarif = 8;           // Tarif étudiant
} else if (age >= 65) {
    tarif = 7;           // Tarif senior
} else {
    tarif = 12;          // Plein tarif
}

console.log(`Tarif : ${tarif} euros`);
// "Tarif : 8 euros"

Conditions imbriquées

On peut aussi mettre un if à l'intérieur d'un autre if. C'est parfois nécessaire, mais attention à la lisibilité : 

const age = 20;
const aPermis = true;
const aVoiture = false;

if (age >= 18) {
    if (aPermis) {
        if (aVoiture) {
            console.log("Tu peux conduire ta voiture");
        } else {
            console.log("Tu as le permis mais pas de voiture");
        }
    } else {
        console.log("Passe d'abord le permis");
    }
} else {
    console.log("Tu es trop jeune");
}
// "Tu as le permis mais pas de voiture"

Conseil : si tu as plus de 2 niveaux d'imbrication, essaie de simplifier avec des opérateurs logiques (&&) ou des fonctions (que tu verras au chapitre 13).

À toi de jouer

  1. Ouvre la console (⌘ + ⌥ + J)
  2. Teste le système de notes avec différentes valeurs
  3. Inverse l'ordre des conditions pour observer le bug
  4. Crée ton propre système de tarification (cinéma, transport, musée...)

La suite

La structure if...else if est très flexible, mais quand tu compares une même variable à de nombreuses valeurs précises (le jour de la semaine, un code d'erreur, un nom de mois), elle devient verbeuse. JavaScript propose une syntaxe plus élégante pour ces cas : l'instruction switch.
