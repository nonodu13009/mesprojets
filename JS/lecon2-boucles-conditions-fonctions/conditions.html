Les conditions if et if...else

Leçon 2 · Chapitre 9 · Prendre des décisions dans le code

Pourquoi des conditions ?

Dans les deux chapitres précédents, tu as appris à répéter du code. Maintenant, tu vas apprendre à faire des choix. En Leçon 1, tu as découvert les opérateurs de comparaison (===, >, <) qui retournent true ou false. Les conditions utilisent ces résultats pour décider quel code exécuter.

C'est comme dire : « Si il pleut, prends un parapluie. Sinon, mets des lunettes de soleil. » Le programme choisit un chemin ou l'autre selon la situation.

La structure if

const age = 20;

if (age >= 18) {
    console.log("Tu es majeur");
}
// "Tu es majeur"

Le code entre les accolades { } ne s'exécute que si la condition entre parenthèses est true. Si elle est false, le bloc est simplement ignoré : 

const age = 15;

if (age >= 18) {
    console.log("Tu es majeur");
}
// Rien ne s'affiche (15 < 18 → condition false)

La structure if...else

Souvent, tu veux exécuter un code alternatif quand la condition est fausse. C'est le rôle de else : 

const age = 15;

if (age >= 18) {
    console.log("Tu es majeur");
} else {
    console.log("Tu es mineur");
}
// "Tu es mineur"
condition true ?→ bloc if condition false ?→ bloc else

Un seul des deux blocs s'exécute — jamais les deux. C'est un embranchement : le programme prend un chemin ou l'autre.

Les opérateurs logiques

On peut combiner plusieurs conditions avec les opérateurs logiques. Tu les utiliseras constamment avec if :

Opérateur Nom Signification Exemple
&& ET (AND) Les deux conditions doivent être vraies age >= 18 && age <= 65
|| OU (OR) Au moins une condition doit être vraie jour === "samedi" || jour === "dimanche"
! NON (NOT) Inverse la condition !estConnecté 
const age = 25;
const aPermis = true;

// ET : les deux doivent être vrais
if (age >= 18 && aPermis) {
    console.log("Tu peux conduire");
}

const jour = "samedi";

// OU : au moins un doit être vrai
if (jour === "samedi" || jour === "dimanche") {
    console.log("C'est le week-end !");
}

const estConnecté = false;

// NON : inverse la valeur
if (!estConnecté) {
    console.log("Veuillez vous connecter");
}

L'opérateur ternaire (raccourci)

Pour les conditions simples qui assignent une valeur, le ternaire permet d'écrire le tout sur une seule ligne : 

// Syntaxe : condition ? valeurSiVrai : valeurSiFaux

const age = 20;
const statut = age >= 18 ? "majeur" : "mineur";
console.log(statut); // "majeur"

// Équivalent à :
let statut2;
if (age >= 18) {
    statut2 = "majeur";
} else {
    statut2 = "mineur";
}

Bonne pratique : utilise le ternaire uniquement pour des conditions simples et courtes. Si c'est complexe, préfère if...else pour la lisibilité. Tu retrouveras le ternaire en Leçon 3 avec les opérateurs modernes.

Exemple concret : formulaire de connexion

const email = "jean@gmail.com";
const motDePasse = "secret123";

if (email === "" || motDePasse === "") {
    console.log("Veuillez remplir tous les champs");
} else if (motDePasse.length < 8) {
    console.log("Le mot de passe doit faire au moins 8 caractères");
} else {
    console.log("Connexion réussie !");
}
// "Connexion réussie !"

Tu remarques qu'on utilise ici else if — c'est justement le sujet du prochain chapitre.

À toi de jouer

  1. Ouvre la console (⌘ + ⌥ + J)
  2. Déclare une variable let age = 15;
  3. Teste : if (age >= 18) { console.log("majeur"); } else { console.log("mineur"); }
  4. Change la valeur de age et relance
  5. Essaie l'opérateur ternaire : age >= 18 ? "majeur" : "mineur"
  6. Combine avec && : if (age >= 18 && age < 65) { ... }

La suite

Tu sais maintenant faire un choix entre deux options. Mais souvent, il y a plus de deux cas possibles : une note peut être « Très bien », « Bien », « Assez bien », « Passable » ou « Insuffisant ». Pour gérer ces situations, on enchaîne plusieurs conditions avec else if.
