Portée, break et continue

Leçon 2 · Chapitre 12 · Comprendre où vivent les variables et contrôler les boucles

Un concept fondamental

Tu as déjà utilisé des variables dans des boucles et des conditions. Mais as-tu remarqué que certaines variables n'existent que à l'intérieur d'un bloc { } ? Ce n'est pas un hasard — c'est la portée (scope).

Comprendre la portée t'évitera une catégorie entière de bugs. Ce chapitre aborde aussi break et continue, deux mots-clés qui te donnent un contrôle fin sur l'exécution des boucles.

La portée des variables (scope)

La portée d'une variable définit où elle est accessible dans le code. Les accolades { } créent un bloc, et les variables déclarées avec let ou const n'existent que dans ce bloc : 

const nom = "Jean-Michel"; // Portée globale

if (true) {
    const age = 30; // Portée locale (bloc if)
    console.log(nom); // "Jean-Michel" → accessible (portée parente)
    console.log(age); // 30 → accessible (même bloc)
}

console.log(nom); // "Jean-Michel" → accessible (portée globale)
// console.log(age); → ERREUR ! age n'existe pas ici

Retiens cette règle : un bloc enfant peut lire les variables de ses parents, mais un parent ne peut pas lire les variables de ses enfants.

Portée dans les boucles

for (let i = 0; i < 3; i++) {
    const message = `Tour ${i}`;
    console.log(message); // Accessible dans la boucle
}

// console.log(i);       → ERREUR ! i n'existe plus
// console.log(message); → ERREUR ! message n'existe plus

C'est pour ça qu'on déclare let i dans le for : le compteur n'a pas besoin d'exister en dehors de la boucle. Si tu as besoin d'une valeur après la boucle, déclare-la avant : 

let total = 0; // Déclarée AVANT la boucle

for (let i = 1; i <= 5; i++) {
    total += i;
}

console.log(total); // 15 → accessible car déclarée en dehors

var vs let/const — pourquoi éviter var

L'ancien mot-clé var ne respecte pas la portée de bloc, ce qui provoque des bugs subtils. C'est la raison principale pour laquelle on utilise let et const en JavaScript moderne : 

if (true) {
    var ancienne = "je fuis le bloc";   // var ignore le bloc !
    let moderne = "je reste dans le bloc";
}

console.log(ancienne); // "je fuis le bloc" → fonctionne (mauvais !)
// console.log(moderne); → ERREUR (bon comportement)
varportée de fonction (à éviter) let / constportée de bloc (à utiliser)

Bonne pratique : utilise const par défaut. Utilise let uniquement si tu dois réassigner la variable. N'utilise jamais var.

break : sortir d'une boucle

Tu as déjà vu break dans le switch. Il fonctionne aussi dans les boucles : il arrête complètement la boucle en cours, immédiatement : 

const nombres = [3, 7, 2, 9, 5, 1];

// Chercher le premier nombre > 8
for (const n of nombres) {
    if (n > 8) {
        console.log(`Trouvé : ${n}`);
        break; // On sort de la boucle
    }
    console.log(`${n} n'est pas > 8`);
}
// 3 n'est pas > 8
// 7 n'est pas > 8
// 2 n'est pas > 8
// Trouvé : 9
// (5 et 1 ne sont jamais testés)

Utilise break quand tu cherches une valeur et que tu veux arrêter dès qu'elle est trouvée — inutile de continuer à parcourir le reste du tableau.

continue : sauter une itération

continue ne sort pas de la boucle — il saute le reste du tour actuel et passe directement au tour suivant : 

// Afficher uniquement les nombres pairs
for (let i = 1; i <= 10; i++) {
    if (i % 2 !== 0) {
        continue; // Nombre impair → on saute au tour suivant
    }
    console.log(i);
}
// 2, 4, 6, 8, 10

Rappel : l'opérateur % (modulo) donne le reste de la division. i % 2 === 0 signifie que i est pair.

break vs continue : résumé

breaksort de la boucle (arrête tout) continuesaute au tour suivant 
// Exemple combiné
const noms = ["Alice", "", "Bob", "STOP", "Charlie"];

for (const nom of noms) {
    if (nom === "STOP") {
        console.log("Arrêt demandé");
        break;       // On arrête tout
    }
    if (nom === "") {
        continue;    // On ignore les noms vides
    }
    console.log(`Bonjour ${nom}`);
}
// Bonjour Alice
// Bonjour Bob
// Arrêt demandé

À toi de jouer

  1. Ouvre la console (⌘ + ⌥ + J)
  2. Déclare une variable dans un if et essaie d'y accéder en dehors
  3. Utilise break dans une boucle pour la stopper à un moment précis
  4. Utilise continue pour filtrer des valeurs dans une boucle
  5. Compare le comportement de var et let dans un bloc
