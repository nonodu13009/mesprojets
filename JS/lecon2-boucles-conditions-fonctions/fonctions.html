Introduction aux fonctions

Leçon 2 · Chapitre 13 · Organiser et réutiliser son code

Le problème de la répétition

Depuis le début de cette leçon, tu écris du code dans des boucles et des conditions. Mais imagine que tu aies besoin du même calcul à plusieurs endroits de ton programme — par exemple, calculer une TVA, valider un email, ou afficher un message formaté. Copier-coller le code à chaque fois pose deux problèmes :

  • Si tu dois le modifier, tu dois le modifier partout
  • Ton code devient long, illisible et difficile à maintenir

Les fonctions résolvent ce problème. Tu écris le code une seule fois, tu lui donnes un nom, et tu l'appelles autant de fois que nécessaire.

Déclarer et appeler une fonction

// Déclaration de fonction
function direBonjour() {
    console.log("Bonjour !");
}

// Appel de la fonction (exécute le code à l'intérieur)
direBonjour(); // "Bonjour !"
direBonjour(); // "Bonjour !" (on peut l'appeler autant de fois qu'on veut)

La déclaration ne fait rien en elle-même — elle prépare le code. C'est l'appel (avec les parenthèses ()) qui l'exécute réellement.

Les paramètres

Les paramètres permettent de passer des valeurs à la fonction pour la rendre flexible. Chaque appel peut donner des valeurs différentes : 

function direBonjour(prénom) {
    console.log(`Bonjour ${prénom} !`);
}

direBonjour("Jean-Michel"); // "Bonjour Jean-Michel !"
direBonjour("Alice");        // "Bonjour Alice !"

// Plusieurs paramètres
function addition(a, b) {
    console.log(`${a} + ${b} = ${a + b}`);
}

addition(3, 5); // "3 + 5 = 8"
addition(10, 20); // "10 + 20 = 30"

Vocabulaire : on parle de paramètres dans la déclaration et d'arguments dans l'appel. En pratique, les deux termes sont souvent utilisés de manière interchangeable.

La valeur de retour (return)

return permet à la fonction de renvoyer un résultat qu'on peut stocker dans une variable ou utiliser dans une expression : 

function addition(a, b) {
    return a + b;
}

const résultat = addition(3, 5);
console.log(résultat); // 8

// On peut utiliser le retour directement
console.log(addition(10, 20)); // 30
console.log(addition(1, 2) + addition(3, 4)); // 10

Important : return arrête immédiatement la fonction. Tout code écrit après un return ne sera jamais exécuté. C'est comme break dans une boucle, mais pour les fonctions.

Paramètres par défaut

Tu peux donner une valeur par défaut à un paramètre. Si l'appel ne fournit pas d'argument, la valeur par défaut est utilisée : 

function direBonjour(prénom = "inconnu") {
    console.log(`Bonjour ${prénom} !`);
}

direBonjour("Alice"); // "Bonjour Alice !"
direBonjour();         // "Bonjour inconnu !" (valeur par défaut)

C'est une fonctionnalité ES6 que tu retrouveras en Leçon 3 avec le destructuring et le rest/spread.

Les fonctions fléchées (arrow functions)

Syntaxe moderne et concise pour écrire des fonctions, introduite avec ES6. Tu les verras partout dans le JavaScript contemporain : 

// Fonction classique
function carré(n) {
    return n * n;
}

// Fonction fléchée équivalente
const carré = (n) => {
    return n * n;
};

// Version ultra-courte (une seule expression → return implicite)
const carré = (n) => n * n;

console.log(carré(4)); // 16
console.log(carré(9)); // 81
Forme Syntaxe Quand l'utiliser
Classique function nom() { } Fonctions principales, hoisting nécessaire
Fléchée const nom = () => { } Callbacks, fonctions courtes
Fléchée courte const nom = (x) => x * 2 Expression unique, retour implicite

Pour l'instant : utilise la forme classique (function) pour tes fonctions principales. Tu adopteras les fonctions fléchées naturellement en Leçon 3 avec les méthodes de tableaux comme map() et filter().

Exemple concret : calculatrice

Cet exemple combine tout ce que tu as appris : fonction, paramètres, return, et switch : 

function calculer(a, b, opération) {
    switch (opération) {
        case "+":
            return a + b;
        case "-":
            return a - b;
        case "*":
            return a * b;
        case "/":
            if (b === 0) {
                return "Erreur : division par zéro";
            }
            return a / b;
        default:
            return "Opération inconnue";
    }
}

console.log(calculer(10, 3, "+")); // 13
console.log(calculer(10, 3, "-")); // 7
console.log(calculer(10, 3, "*")); // 30
console.log(calculer(10, 3, "/")); // 3.333...
console.log(calculer(10, 0, "/")); // "Erreur : division par zéro"

Exemple : fonction qui filtre un tableau

Les fonctions et les boucles se combinent parfaitement. Voici une fonction qui prend un tableau et retourne un nouveau tableau contenant uniquement les nombres pairs : 

function filtrerPairs(nombres) {
    const pairs = [];
    for (const n of nombres) {
        if (n % 2 === 0) {
            pairs.push(n);
        }
    }
    return pairs;
}

const mesNombres = [1, 2, 3, 4, 5, 6, 7, 8];
console.log(filtrerPairs(mesNombres));
// [2, 4, 6, 8]

En Leçon 3, tu découvriras que cette opération peut s'écrire en une seule ligne avec mesNombres.filter(n => n % 2 === 0). Mais pour l'instant, cette version « manuelle » te permet de bien comprendre le mécanisme.

À toi de jouer

  2. Crée une fonction function bonjour(nom) { return "Salut " + nom; }
  3. Appelle-la : bonjour("Alice")
  4. Teste une fonction fléchée : const double = (n) => n * 2;
  5. Crée la calculatrice et teste toutes les opérations
  6. Écris une fonction qui retourne le plus grand nombre d'un tableau
