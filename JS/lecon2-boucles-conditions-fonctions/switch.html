L'instruction switch

Leçon 2 · Chapitre 11 · Simplifier les comparaisons multiples

Une alternative à else if

Au chapitre précédent, tu as enchaîné des else if pour gérer plusieurs cas. Mais quand tu compares une même variable à plusieurs valeurs exactes (pas des intervalles), le code devient répétitif : 

// Avec if...else if (répétitif) :
if (jour === "lundi") { ... }
else if (jour === "mardi") { ... }
else if (jour === "mercredi") { ... }
// ...

Le switch offre une syntaxe plus claire pour ce type de situation. Il liste les cas possibles de manière structurée.

Syntaxe du switch

const jour = "lundi";

switch (jour) {
    case "lundi":
        console.log("Début de semaine");
        break;
    case "mercredi":
        console.log("Milieu de semaine");
        break;
    case "vendredi":
        console.log("Bientôt le week-end !");
        break;
    case "samedi":
    case "dimanche":
        console.log("Week-end !");
        break;
    default:
        console.log("Jour ordinaire");
}
// "Début de semaine"

Les éléments clés

Mot-clé Rôle
switch(expr) L'expression à tester (souvent une variable)
case valeur: Compare avec expr === valeur (égalité stricte)
break Sort du switch — obligatoire après chaque cas !
default S'exécute si aucun case ne correspond (comme else)

Attention : le switch utilise la comparaison stricte (===). case 5 ne correspondra pas à la chaîne "5". Tu retrouves ici l'importance du === vu en Leçon 1.

Le piège du break oublié

Sans break, le code « tombe » dans les case suivants. On appelle ça le fall-through — c'est presque toujours un bug : 

const fruit = "pomme";

// SANS break → Fall-through !
switch (fruit) {
    case "pomme":
        console.log("C'est une pomme");
    case "banane":
        console.log("C'est une banane");
    case "cerise":
        console.log("C'est une cerise");
}
// Affiche les TROIS lignes ! (pas le comportement voulu)

Règle d'or : mets TOUJOURS un break à la fin de chaque case, sauf si tu regroupe volontairement des cas (voir ci-dessous).

Regrouper des cas

Le fall-through est utile quand plusieurs valeurs doivent produire le même résultat. On empile les case sans break entre eux : 

const mois = 3; // Mars

switch (mois) {
    case 12:
    case 1:
    case 2:
        console.log("Hiver");
        break;
    case 3:
    case 4:
    case 5:
        console.log("Printemps");
        break;
    case 6:
    case 7:
    case 8:
        console.log("Été");
        break;
    case 9:
    case 10:
    case 11:
        console.log("Automne");
        break;
    default:
        console.log("Mois invalide");
}
// "Printemps"

switch vs if...else if : que choisir ?

Critère switch if...else if
Comparer à des valeurs précises Idéal Possible mais verbeux
Comparer avec des intervalles (>, <) Impossible Idéal
Beaucoup de cas (5+) Plus lisible Verbeux
Conditions complexes (&&, ||) Impossible Idéal

En résumé : utilise switch quand tu compares une variable à une liste de valeurs fixes. Utilise if...else if pour tout le reste.

À toi de jouer

  1. Ouvre la console (⌘ + ⌥ + J)
  2. Crée une variable const jour = "mardi";
  3. Écris un switch sur les jours de la semaine
  4. Enlève un break pour observer le fall-through
  5. Teste le switch des saisons avec différents mois
