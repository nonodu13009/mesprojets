Les opérateurs de comparaison

Leçon 1 · Chapitre 5 · Comparer des valeurs en JavaScript

Pourquoi comparer ?

Tu sais maintenant créer des variables et manipuler du texte. Mais un programme a souvent besoin de prendre des décisions : est-ce que l'utilisateur est majeur ? Est-ce que le mot de passe est correct ? Est-ce que le panier dépasse 100€ ?

Les opérateurs de comparaison permettent de comparer deux valeurs entre elles. Le résultat est toujours un booléen : true (vrai) ou false (faux). En leçon 2, tu utiliseras ces résultats dans des conditions if/else.

Les opérateurs de comparaison

Opérateur Signification Exemple Résultat
== Égal à (valeur) 5 == "5" true
=== Strictement égal (valeur + type) 5 === "5" false
!= Différent de (valeur) 5 != "5" false
!== Strictement différent (valeur + type) 5 !== "5" true
> Supérieur à 10 > 5 true
< Inférieur à 10 < 5 false
>= Supérieur ou égal à 5 >= 5 true
<= Inférieur ou égal à 3 <= 5 true

== vs === : la différence cruciale

C'est le piège classique en JavaScript. Tu te rappelles les types du chapitre 3 ? C'est ici qu'ils deviennent importants. L'opérateur == compare uniquement la valeur en convertissant les types si besoin. L'opérateur === compare la valeur ET le type, sans conversion. 

// == fait de la conversion de type (coercion)
console.log(5 == "5");    // true  (le string "5" est converti en number)
console.log(0 == false);   // true  (false est converti en 0)
console.log("" == false);  // true  (les deux sont convertis en 0)

// === ne convertit rien : valeur ET type doivent correspondre
console.log(5 === "5");   // false (number vs string)
console.log(0 === false);  // false (number vs boolean)
console.log("" === false); // false (string vs boolean)

Bonne pratique : utilise TOUJOURS === et !==. L'égalité lâche (==) peut donner des résultats surprenants et est source de bugs difficiles à trouver.

Comparer des chaînes

Les chaînes sont comparées caractère par caractère selon l'ordre alphabétique (plus précisément l'ordre Unicode). Attention à la casse : les majuscules et minuscules ne sont pas équivalentes. 

console.log("a" < "b");     // true  ("a" vient avant "b")
console.log("Z" < "a");     // true  (les majuscules viennent avant)

// Attention à la casse !
console.log("Bonjour" === "bonjour"); // false

// Pour comparer sans tenir compte de la casse :
const mot1 = "Bonjour";
const mot2 = "bonjour";
console.log(mot1.toLowerCase() === mot2.toLowerCase()); // true

Exemple concret : vérification d'âge

Voici un aperçu de ce que tu feras en leçon 2 avec les conditions if/else. Les comparaisons sont la base de toute prise de décision dans le code : 

const age = 17;

if (age >= 18) {
    console.log("Accès autorisé");
} else {
    console.log(`Accès refusé, tu as ${age} ans`);
}
// "Accès refusé, tu as 17 ans"

À toi de jouer

  1. Ouvre la console du navigateur (F12 → Console)
  2. Tape 5 === "5" et observe le résultat (false)
  3. Compare avec 5 == "5" (true) — sens-tu la différence ?
  4. Teste des comparaisons de nombres : 10 > 5, 3 <= 3
  5. Essaie avec des chaînes : "abc" < "abd"
