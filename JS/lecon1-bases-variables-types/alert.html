Les alertes système

Leçon 1 · Chapitre 2 · alert, confirm et prompt

Communiquer avec l'utilisateur

Au chapitre précédent, tu as utilisé console.log() pour afficher des messages. Mais ces messages n'apparaissent que dans la console de développement — l'utilisateur normal ne les voit pas.

JavaScript propose trois boîtes de dialogue natives du navigateur pour interagir directement avec l'utilisateur. Elles sont simples, un peu "brutes", mais parfaites pour apprendre et tester rapidement du code.

Les 3 types d'alertes

Méthode Rôle Retourne
alert() Affiche un message undefined
confirm() Pose une question OK / Annuler true / false
prompt() Demande une saisie texte string / null

1. alert() — afficher un message

La plus simple : elle affiche un message et attend que l'utilisateur clique sur OK. Elle ne retourne rien (undefined). On l'utilise pour informer. 

alert("Bienvenue sur le site !");

2. confirm() — poser une question

Affiche un message avec deux boutons : OK et Annuler. Elle retourne true si OK, false si Annuler. C'est ta première interaction qui produit un résultat utilisable dans le code. 

const réponse = confirm("Voulez-vous continuer ?");

console.log(réponse); // true ou false

if (réponse) {
  console.log("L'utilisateur a dit oui");
} else {
  console.log("L'utilisateur a dit non");
}

3. prompt() — demander une saisie

Affiche un champ de texte. Elle retourne la valeur saisie (un string), ou null si l'utilisateur annule. C'est la seule alerte qui permet de récupérer une donnée de l'utilisateur. 

const nom = prompt("Comment tu t'appelles ?");

if (nom !== null) {
  console.log("Bonjour " + nom);
} else {
  console.log("L'utilisateur a annulé");
}

On peut aussi passer une valeur par défaut en second argument :

const ville = prompt("Ta ville ?", "Marseille");

Récapitulatif visuel

alert( message )information confirm( question )oui / non prompt( question )saisie texte

Pourquoi éviter les alertes système ?

Ces boîtes de dialogue sont pratiques pour apprendre, mais on les évite dans les vrais projets pour plusieurs raisons :

  • Elles bloquent toute interaction avec la page
  • Leur apparence dépend du navigateur — on ne peut pas les styliser
  • Elles offrent une mauvaise expérience utilisateur

En pratique, on les remplace par des composants HTML/CSS personnalisés (modales, toasts, notifications). Mais pour débuter et tester rapidement du code, elles sont parfaites.

La suite

Tu sais maintenant afficher des messages et récupérer des données de l'utilisateur. Mais tu as remarqué que prompt() retourne toujours un string, même si l'utilisateur tape un nombre. Pour comprendre pourquoi, il faut connaître les types de valeurs en JavaScript — c'est le sujet du prochain chapitre.
