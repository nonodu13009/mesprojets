Les types de valeurs

Leçon 1 · Chapitre 3 · Les 7 types fondamentaux de JavaScript

Pourquoi les types comptent

Au chapitre précédent, tu as vu que prompt() retourne toujours un string, même quand l'utilisateur tape un nombre. Si tu fais "5" + 3, JavaScript te donnera "53" (concaténation de texte) au lieu de 8 (addition).

Pour éviter ce genre de surprise, il faut comprendre les types de valeurs : chaque donnée en JavaScript appartient à un type, et le type détermine comment JavaScript la traite.

Deux grandes familles

En JavaScript, il existe deux familles : les types primitifs (simples, immuables) et les types objets (complexes, modifiables).

Primitifs (6)valeurs simples Object (1)valeurs complexes

Les 6 types primitifs

Type Description Exemple
string Chaîne de caractères (texte) "Bonjour"
number Nombre entier ou décimal 42, 3.14
boolean Vrai ou faux true, false
undefined Variable déclarée mais sans valeur undefined
null Absence intentionnelle de valeur null
symbol Identifiant unique (avancé) Symbol("id")

Les trois premiers (string, number, boolean) sont ceux que tu utiliseras 90% du temps. Les autres sont plus spécifiques.

undefined vs null — la différence

Ces deux types représentent "pas de valeur", mais pour des raisons différentes. Imagine un formulaire avec un champ "prénom" :

undefined null
Qui décide ? JavaScript (automatique) Le développeur (volontaire)
Signification "Aucune valeur n'a été donnée" "La valeur est volontairement vide"
Analogie Un champ de formulaire jamais rempli Un champ effacé volontairement par l'utilisateur 
// undefined — JavaScript le met automatiquement
let prénom;
console.log(prénom); // undefined (déclarée, mais pas de valeur)

// null — c'est le développeur qui le décide
let utilisateur = null;
console.log(utilisateur); // null (on dit "pas d'utilisateur pour l'instant")

En résumé : undefined = "personne n'a mis de valeur ici". null = "quelqu'un a décidé que la valeur est vide". Dans le doute, utilise === null ou === undefined (triple égal) pour les distinguer.

Le type Object

Tout ce qui n'est pas primitif est un objet. Les objets sont des collections de paires clé/valeur. Plusieurs structures connues sont des objets :

Structure Description Exemple
Object Collection de propriétés { nom: "Alice" }
Array Liste ordonnée de valeurs [1, 2, 3]
Function Bloc de code réutilisable function() {}
Date Date et heure new Date()

typeof — connaître le type d'une valeur

L'opérateur typeof retourne une chaîne indiquant le type d'une valeur. C'est ton outil de diagnostic quand quelque chose se comporte de façon inattendue : 

console.log(typeof "Bonjour");   // "string"
console.log(typeof 42);           // "number"
console.log(typeof true);         // "boolean"
console.log(typeof undefined);    // "undefined"
console.log(typeof null);         // "object" (bug historique !)
console.log(typeof [1, 2]);       // "object"
console.log(typeof {});           // "object"

Piège classique : typeof null retourne "object". C'est un bug historique de JavaScript qui n'a jamais été corrigé pour ne pas casser le web existant.

Primitif vs Objet — la différence clé

Cette distinction aura de l'importance plus tard, quand tu manipuleras des objets complexes. Pour l'instant, retiens l'essentiel :

Primitif Objet
Modifiable ? Non (immuable) Oui (modifiable)
Copie Duplique la valeur Copie la référence 
// Primitif : copie indépendante
let a = 5;
let b = a;
b = 10;
console.log(a); // 5 (inchangé)

// Objet : même référence
let obj1 = { nom: "Alice" };
let obj2 = obj1;
obj2.nom = "Bob";
console.log(obj1.nom); // "Bob" (modifié aussi !)

À toi de jouer

  1. Ouvre la console du navigateur (F12 → Console)
  2. Tape typeof "hello" et appuie sur Entrée
  3. Essaie avec différentes valeurs : typeof 42, typeof true, typeof null
  4. Teste le piège : typeof null === "object"
