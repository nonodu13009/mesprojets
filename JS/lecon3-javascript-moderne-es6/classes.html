Les classes ES6

Leçon 3 · Chapitre 27 · Créer des objets avec un plan de construction

Des objets, oui, mais structurés

Jusqu'ici, tu as créé des objets "à la volée" avec la syntaxe littérale { nom: "...", age: 30 }. Ça marche très bien pour un ou deux objets, mais imagine que tu doives créer 50 utilisateurs, ou 200 produits. Tu vas copier-coller la même structure à chaque fois ?

Les classes résolvent ce problème. Une classe est un modèle (un "plan de construction") qui décrit la structure et les comportements que partageront tous les objets créés à partir d'elle. Pense à un emporte-pièce : la classe est le moule, et les objets sont les biscuits.

Ta première classe

Commençons par un exemple simple : une classe Animal. On la déclare avec le mot-clé class, puis on définit un constructeur (la fonction qui initialise l'objet) et des méthodes (les actions que l'objet peut faire) : 

class Animal {
    // Le constructeur est appelé automatiquement par "new"
    constructor(nom, type) {
        this.nom = nom;
        this.type = type;
    }

    // Une méthode (pas besoin du mot-clé "function")
    sePresenter() {
        console.log(`Je suis ${this.nom}, un ${this.type}.`);
    }
}

// Créer des instances avec le mot-clé "new"
const chat = new Animal("Minou", "chat");
const chien = new Animal("Rex", "chien");

chat.sePresenter();  // "Je suis Minou, un chat."
chien.sePresenter(); // "Je suis Rex, un chien."

console.log(chat.nom); // "Minou"

Chaque appel à new Animal(...) crée un objet indépendant avec ses propres valeurs, mais qui partage les mêmes méthodes. C'est ça la puissance d'une classe : une seule définition, des dizaines d'objets.

Le mot-clé this : dans une méthode de classe, this fait référence à l'instance courante. Quand tu appelles chat.sePresenter(), this représente chat. Quand tu appelles chien.sePresenter(), this représente chien.

Un exemple plus concret : compte bancaire

Les classes deviennent vraiment utiles quand l'objet a un état interne qui change au fil du temps. Un compte bancaire est un bon exemple : il a un solde qui augmente ou diminue : 

class CompteBancaire {
    constructor(titulaire, solde = 0) {
        this.titulaire = titulaire;
        this.solde = solde;
    }

    déposer(montant) {
        this.solde += montant;
        console.log(`+${montant}€ → Solde : ${this.solde}€`);
    }

    retirer(montant) {
        if (montant > this.solde) {
            console.log("Solde insuffisant !");
            return;
        }
        this.solde -= montant;
        console.log(`-${montant}€ → Solde : ${this.solde}€`);
    }

    afficher() {
        console.log(`Compte de ${this.titulaire} : ${this.solde}€`);
    }
}

const compte = new CompteBancaire("Jean-Michel", 1000);
compte.déposer(500);  // "+500€ → Solde : 1500€"
compte.retirer(200);  // "-200€ → Solde : 1300€"
compte.retirer(5000); // "Solde insuffisant !"
compte.afficher();     // "Compte de Jean-Michel : 1300€"

Remarque comment les méthodes déposer et retirer modifient this.solde. L'objet "se souvient" de son état entre chaque appel. C'est ce qui rend les classes si utiles pour modéliser des choses du monde réel.

Héritage : créer des classes à partir d'autres classes

Parfois, plusieurs classes partagent des propriétés communes. Un chien et un chat sont tous les deux des animaux — ils ont un nom et peuvent "parler". Plutôt que de dupliquer ce code, on peut faire hériter une classe d'une autre avec le mot-clé extends : 

class Animal {
    constructor(nom) {
        this.nom = nom;
    }

    parler() {
        console.log(`${this.nom} fait du bruit.`);
    }
}

class Chien extends Animal {
    constructor(nom, race) {
        super(nom); // Appelle le constructeur du parent (Animal)
        this.race = race;
    }

    // On peut redéfinir une méthode du parent
    parler() {
        console.log(`${this.nom} aboie ! Wouf !`);
    }

    afficherRace() {
        console.log(`${this.nom} est un ${this.race}.`);
    }
}

const rex = new Chien("Rex", "Berger Allemand");
rex.parler();       // "Rex aboie ! Wouf !"
rex.afficherRace(); // "Rex est un Berger Allemand."

super() est obligatoire dans le constructeur d'une classe enfant. Il appelle le constructeur du parent pour initialiser les propriétés héritées. Il doit être appelé avant toute utilisation de this.

Getters : des propriétés calculées

Parfois, tu veux une propriété dont la valeur est calculée à partir d'autres propriétés (comme la surface d'un cercle à partir de son rayon). Les getters permettent de définir une méthode qui s'utilise comme une propriété — sans parenthèses : 

class Cercle {
    constructor(rayon) {
        this.rayon = rayon;
    }

    get surface() {
        return Math.PI * this.rayon ** 2;
    }

    get périmètre() {
        return 2 * Math.PI * this.rayon;
    }
}

const c = new Cercle(5);
console.log(c.surface);   // 78.539... (pas de parenthèses !)
console.log(c.périmètre); // 31.415...

On écrit c.surface et non c.surface(). Pour celui qui utilise l'objet, ça ressemble à une propriété normale, mais en réalité c'est une valeur recalculée à chaque accès.
