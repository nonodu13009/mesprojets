Les Méthodes des Objets

Leçon 3 · Chapitre 27 · Donner des comportements à tes objets

C'est quoi, une méthode ?

Jusqu'ici, tes objets contenaient des données : chaînes, nombres, booléens. Mais une valeur peut aussi être une fonction. Une fonction stockée dans un objet s'appelle une méthode. 

const utilisateur = {
    prenom: "Jean-Michel",
    nom: "Nogaro",
    saluer: function() {
        console.log("Bonjour !");
    }
};

// On appelle une méthode avec le point + parenthèses
utilisateur.saluer(); // "Bonjour !"

Tu en utilises déjà sans le savoir : console.log() est la méthode log de l'objet console, et "abc".toUpperCase() est une méthode des chaînes.

La syntaxe raccourcie ES6

ES6 permet d'écrire les méthodes sans le mot-clé function. C'est la forme que tu verras partout dans le code moderne : 

const utilisateur = {
    prenom: "Jean-Michel",

    // Avant ES6
    saluerAncien: function() {
        console.log("Bonjour !");
    },

    // Depuis ES6 : plus court, même résultat
    saluer() {
        console.log("Bonjour !");
    }
};

Le mot-clé this

L'intérêt d'une méthode, c'est qu'elle peut accéder aux autres propriétés de son objet grâce au mot-clé this ("cet objet-ci") : 

const utilisateur = {
    prenom: "Jean-Michel",
    nom: "Nogaro",

    nomComplet() {
        return `${this.prenom} ${this.nom}`;
    },

    sePresenter() {
        // Une méthode peut appeler une autre méthode du même objet
        console.log(`Bonjour, je suis ${this.nomComplet()}`);
    }
};

console.log(utilisateur.nomComplet()); // "Jean-Michel Nogaro"
utilisateur.sePresenter();             // "Bonjour, je suis Jean-Michel Nogaro"

Quand tu appelles utilisateur.nomComplet(), this désigne l'objet avant le point, donc utilisateur.

Piège : pas de fléchée pour les méthodes

Tu te souviens du chapitre sur les fonctions fléchées ? Elles n'ont pas leur propre this. Résultat : une méthode écrite en fléchée ne peut pas accéder aux propriétés de son objet. 

const compteur = {
    valeur: 10,

    // ❌ Fléchée : this ne désigne PAS l'objet compteur
    afficherMal: () => {
        console.log(this.valeur); // undefined
    },

    // ✅ Syntaxe raccourcie : this fonctionne
    afficherBien() {
        console.log(this.valeur); // 10
    }
};

Règle à retenir : fléchées pour les callbacks, syntaxe raccourcie nomMethode() { } pour les méthodes d'objet qui utilisent this.

Exemple concret : un panier d'achat

Combinons données et comportements dans un objet réaliste : 

const panier = {
    articles: [],

    ajouter(nom, prix) {
        this.articles.push({ nom, prix }); // raccourci d'objet du chap. 25 !
        console.log(`${nom} ajouté au panier`);
    },

    total() {
        let somme = 0;
        for (const article of this.articles) {
            somme += article.prix;
        }
        return somme;
    }
};

panier.ajouter("Clavier", 49);  // "Clavier ajouté au panier"
panier.ajouter("Souris", 29);   // "Souris ajouté au panier"
console.log(panier.total());      // 78

C'est le principe de base de la programmation orientée objet : regrouper au même endroit les données (articles) et les actions qui les manipulent (ajouter, total).

À toi de jouer

  1. Ouvre la console (F12 → Console)
  2. Crée un objet chien avec une propriété nom et une méthode aboyer() qui affiche "Wouf !"
  3. Ajoute une méthode sePresenter() qui utilise this.nom
  4. Teste la même méthode en fléchée et observe le undefined
  5. Recrée le panier d'achat et ajoute une méthode vider() qui remet this.articles à []
