Spread et Rest

Leçon 3 · Chapitre 25 · Les trois petits points magiques ...

Le lien avec la destructuration

Au chapitre précédent, tu as appris à extraire des valeurs depuis un objet ou un tableau. Mais tu t'es peut-être demandé : "Et si je veux extraire les deux premières valeurs et récupérer tout le reste d'un coup ?"

C'est exactement ce que permet l'opérateur ... (trois points). Ce petit symbole a deux visages selon le contexte, et une fois que tu le maîtrises, tu le verras partout dans le JavaScript moderne.

Un seul symbole, deux rôles

Le même opérateur ... fait deux choses opposées selon l'endroit où tu l'utilises :

Spreadétaler / décompresser Restrassembler / collecter
Opérateur Contexte Ce qu'il fait
... Spread Dans un tableau ou un objet qu'on construit Étale les éléments un par un
... Rest Dans les paramètres d'une fonction ou une destructuration Rassemble les éléments restants

Ne t'inquiète pas si ça semble abstrait. Les exemples qui suivent vont rendre ça très concret.

Spread : étaler un tableau

Imagine que tu as deux tableaux et que tu veux les fusionner en un seul. Avant ES6, il fallait utiliser concat(). Avec le spread, c'est beaucoup plus intuitif — tu "déverses" le contenu d'un tableau dans un autre : 

const fruits = ["pomme", "banane"];
const légumes = ["carotte", "poireau"];

// Fusionner deux tableaux
const aliments = [...fruits, ...légumes];
console.log(aliments);
// ["pomme", "banane", "carotte", "poireau"]

// Copier un tableau (copie indépendante)
const copieFruits = [...fruits];
copieFruits.push("cerise");

console.log(fruits);      // ["pomme", "banane"] → l'original est intact
console.log(copieFruits); // ["pomme", "banane", "cerise"]

// Ajouter des éléments au milieu
const plusDeFruits = ["kiwi", ...fruits, "mangue"];
console.log(plusDeFruits);
// ["kiwi", "pomme", "banane", "mangue"]

Pense au spread comme à quelqu'un qui ouvre un sac et en sort tous les éléments un par un pour les mettre dans un nouveau sac.

Spread : étaler un objet

Ça fonctionne exactement pareil avec les objets. C'est particulièrement utile pour créer une copie enrichie ou modifiée d'un objet : 

const base = { nom: "Nogaro", ville: "Marseille" };

// Copier et enrichir un objet
const complet = { ...base, age: 30, métier: "Développeur" };
console.log(complet);
// { nom: "Nogaro", ville: "Marseille", age: 30, métier: "Développeur" }

// Écraser une propriété
const modifié = { ...base, ville: "Paris" };
console.log(modifié);
// { nom: "Nogaro", ville: "Paris" }

L'ordre compte : si une propriété apparaît plusieurs fois, c'est la dernière qui gagne. C'est pour ça qu'on met ...base en premier et les modifications après.

Rest : rassembler les arguments restants

Maintenant, passons à l'autre visage du .... Quand il est utilisé dans les paramètres d'une fonction, il fait le contraire du spread : il rassemble tous les arguments restants dans un tableau. 

function additionner(premier, ...autres) {
    console.log(premier); // 1
    console.log(autres);  // [2, 3, 4, 5]

    let total = premier;
    for (const n of autres) {
        total += n;
    }
    return total;
}

console.log(additionner(1, 2, 3, 4, 5)); // 15

Le premier argument va dans premier, et tous les suivants sont "ramassés" dans le tableau autres. C'est très pratique quand tu ne sais pas à l'avance combien d'arguments ta fonction recevra.

Règle : le paramètre rest (...autres) doit toujours être le dernier paramètre de la fonction. function(a, ...b, c) provoquera une erreur.

Rest + destructuration : le combo

Tu te rappelles la destructuration du chapitre précédent ? Le rest s'y intègre parfaitement. Tu peux extraire certaines valeurs et récupérer le reste d'un coup : 

// Avec un tableau
const [premier, deuxième, ...reste] = [1, 2, 3, 4, 5];

console.log(premier);  // 1
console.log(deuxième); // 2
console.log(reste);    // [3, 4, 5]

// Avec un objet
const { nom, ...infos } = { nom: "Nogaro", age: 30, ville: "Marseille" };

console.log(nom);   // "Nogaro"
console.log(infos); // { age: 30, ville: "Marseille" }

C'est exactement la question qu'on posait en intro : "extraire certaines valeurs et récupérer tout le reste". Destructuration + rest = réponse.

Exemple concret : fusionner des configurations

Un cas d'usage très courant en vrai projet : tu as des paramètres par défaut et l'utilisateur peut les surcharger. Le spread rend ça élégant : 

const configDéfaut = {
    thème: "dark",
    langue: "fr",
    notifications: true,
    fontSize: 14
};

const configUtilisateur = {
    thème: "light",
    fontSize: 18
};

// Les préférences utilisateur écrasent les valeurs par défaut
const config = { ...configDéfaut, ...configUtilisateur };

console.log(config);
// { thème: "light", langue: "fr", notifications: true, fontSize: 18 }

thème et fontSize sont écrasés par les préférences de l'utilisateur, mais langue et notifications gardent leurs valeurs par défaut. Exactement le comportement attendu.

À toi de jouer

  1. Ouvre la console (F12 → Console)
  2. Crée deux tableaux et fusionne-les avec [...tab1, ...tab2]
  3. Copie un objet avec { ...monObjet } et modifie la copie sans toucher l'original
  4. Crée une fonction avec ...args et affiche tous les arguments
  5. Teste la destructuration avec rest : const [a, ...b] = [1, 2, 3, 4]
