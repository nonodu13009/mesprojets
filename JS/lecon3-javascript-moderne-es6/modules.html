Les modules

Leçon 3 · Chapitre 29 · Organiser son code en fichiers séparés

Le problème : tout dans un seul fichier

Jusqu'ici, tout ton code JavaScript tenait dans un seul fichier script.js. Pour les exercices et les petits projets, ça fonctionne très bien. Mais imagine un vrai projet avec des centaines de fonctions, des classes, des utilitaires... Tout mettre dans un seul fichier de 3000 lignes devient vite ingérable.

Les modules ES6 permettent de découper ton code en fichiers indépendants, chacun avec ses propres responsabilités. Un fichier s'occupe des utilisateurs, un autre des calculs, un autre de l'affichage. Chaque fichier exporte ce qu'il veut partager, et les autres fichiers importent ce dont ils ont besoin.

Ce système est à la base de tous les frameworks modernes (React, Vue, Angular, Next.js). Comprendre les modules, c'est comprendre comment les vrais projets sont organisés.

Export nommé (named export)

La première façon d'exporter : on met le mot-clé export devant chaque élément qu'on veut rendre disponible. C'est l'export nommé, car chaque export a un nom précis : 

// fichier : maths.js

export const PI = 3.14159;

export function carré(n) {
    return n * n;
}

export function cube(n) {
    return n ** 3;
}

Pour utiliser ces exports dans un autre fichier, on les importe avec des accolades { }. Tu remarqueras que c'est la même syntaxe que la destructuration — ce n'est pas un hasard, l'idée est la même : extraire des éléments spécifiques. 

// fichier : app.js

import { PI, carré, cube } from "./maths.js";

console.log(PI);       // 3.14159
console.log(carré(4)); // 16
console.log(cube(3));  // 27

Export par défaut (default export)

Chaque fichier peut avoir un seul export par défaut. C'est l'élément principal du module — souvent une classe ou une fonction centrale : 

// fichier : Utilisateur.js

export default class Utilisateur {
    constructor(nom, email) {
        this.nom = nom;
        this.email = email;
    }

    sePresenter() {
        console.log(`${this.nom} (${this.email})`);
    }
}

Pour importer un export par défaut, pas d'accolades. Et tu peux lui donner le nom que tu veux : 

// fichier : app.js

import Utilisateur from "./Utilisateur.js";

const u = new Utilisateur("Jean-Michel", "jm@test.com");
u.sePresenter();

Combiner les deux

Un même fichier peut avoir un export par défaut et des exports nommés. C'est très courant en pratique : 

// fichier : utils.js

export default function saluer(nom) {
    return `Bonjour ${nom} !`;
}

export const VERSION = "1.0.0";
export const AUTEUR = "Jean-Michel";
// fichier : app.js

import saluer, { VERSION, AUTEUR } from "./utils.js";

console.log(saluer(AUTEUR)); // "Bonjour Jean-Michel !"
console.log(VERSION);        // "1.0.0"

Le défaut est importé sans accolades, les nommés sont entre accolades. Les deux cohabitent dans la même ligne d'import.

Renommer avec as

Si un nom d'import entre en conflit avec une variable existante, ou si tu veux un nom plus explicite, tu peux renommer avec as : 

import { carré as square, cube as cubic } from "./maths.js";

console.log(square(4)); // 16
console.log(cubic(3));  // 27

// Tout importer dans un objet "namespace"
import * as Maths from "./maths.js";

console.log(Maths.PI);       // 3.14159
console.log(Maths.carré(5)); // 25

Utiliser les modules dans le navigateur

Pour que le navigateur comprenne les import/export, il faut ajouter l'attribut type="module" à la balise <script> : 

<!-- Dans le HTML -->
<script type="module" src="app.js"></script>

Important : les modules ne fonctionnent pas en ouvrant le fichier HTML directement (file://). Il faut un serveur local (par exemple l'extension Live Server de VS Code). C'est une restriction de sécurité du navigateur.

Résumé

Type Export Import
Nommé export const x = ... import { x } from "..."
Par défaut export default ... import x from "..."
Tout — import * as X from "..."
Renommé — import { x as y } from "..."

À toi de jouer

  1. Crée deux fichiers : maths.js et app.js
  2. Dans maths.js, exporte une fonction et une constante
  3. Dans app.js, importe-les et utilise-les
  4. Dans le HTML, utilise <script type="module" src="app.js">
  5. Lance un serveur local (Live Server) et ouvre la console
