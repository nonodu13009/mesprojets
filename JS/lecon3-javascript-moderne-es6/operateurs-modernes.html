Opérateurs modernes

Leçon 3 · Chapitre 30 · Ternaire, optional chaining et nullish coalescing

Les petits raccourcis qui changent tout

Ce dernier chapitre te présente trois opérateurs que tu vas croiser constamment dans le code JavaScript moderne. Ils ne sont pas aussi "gros" que les classes ou les promesses, mais ils rendent ton code plus concis et plus sûr au quotidien.

Tu connais déjà les conditions if/else de la leçon 2. Ici, tu vas découvrir des façons plus courtes d'exprimer la même logique, et surtout deux opérateurs récents (?. et ??) qui résolvent un problème très fréquent : accéder à des données qui pourraient ne pas exister.

L'opérateur ternaire ? :

Le ternaire est un raccourci pour un if/else simple. Au lieu de 5 lignes, tu écris une seule expression. La syntaxe : condition ? valeurSiVrai : valeurSiFaux 

const age = 20;

// Avec if/else (5 lignes)
let statut;
if (age >= 18) {
    statut = "majeur";
} else {
    statut = "mineur";
}

// Avec le ternaire (1 ligne)
const statut = age >= 18 ? "majeur" : "mineur";
console.log(statut); // "majeur"

Le ternaire est particulièrement utile dans les template literals et les assignations de variables : 

// Dans un template literal
console.log(`Tu es ${age >= 18 ? "majeur" : "mineur"}`);

// Pour choisir une valeur
const prix = 150;
const livraison = prix > 100 ? "gratuite" : "5€";
console.log(livraison); // "gratuite"

Bonne pratique : utilise le ternaire uniquement pour des choix simples en une ligne. Si la logique est complexe ou imbriquée, un if/else classique sera plus lisible.

Optional chaining ?.

Voici un problème que tu rencontreras très vite dans un vrai projet : tu essaies d'accéder à une propriété profonde d'un objet, mais un des éléments intermédiaires n'existe pas. Résultat : crash. 

const utilisateur = {
    nom: "Jean-Michel",
    adresse: {
        ville: "Marseille",
        codePostal: "13008"
    }
};

// Ça fonctionne
console.log(utilisateur.adresse.ville); // "Marseille"

// Mais si l'utilisateur n'a pas d'adresse...
const utilisateur2 = { nom: "Alice" };

// ERREUR ! Cannot read property 'ville' of undefined
// console.log(utilisateur2.adresse.ville);

Avant l'optional chaining, il fallait vérifier chaque niveau avec un if ou un &&. C'était lourd. L'opérateur ?. résout ça élégamment : si l'élément avant le ?. est null ou undefined, il retourne undefined au lieu de crasher : 

// Avec optional chaining — retourne undefined au lieu de crasher
console.log(utilisateur2.adresse?.ville); // undefined (pas d'erreur !)

// On peut chaîner
console.log(utilisateur2.adresse?.codePostal?.toUpperCase()); // undefined

// Fonctionne aussi avec les méthodes
const résultat = utilisateur.getAge?.();
console.log(résultat); // undefined (la méthode n'existe pas, pas d'erreur)

// Et avec les tableaux
const liste = null;
console.log(liste?.[0]); // undefined (pas d'erreur)

Nullish coalescing ??

Maintenant qu'on sait que ?. retourne undefined quand quelque chose n'existe pas, on a besoin d'un moyen de fournir une valeur par défaut. C'est le rôle de ?? : il retourne l'opérande de droite quand celui de gauche est null ou undefined. 

const nom = null;
console.log(nom ?? "Anonyme"); // "Anonyme"

const prénom = undefined;
console.log(prénom ?? "Inconnu"); // "Inconnu"

const age = 0;
console.log(age ?? 18); // 0 (0 n'est ni null ni undefined !)

"Mais attends, on avait déjà || pour ça en leçon 2 !" Oui, mais || a un piège : 

const score = 0;

console.log(score || 10); // 10 — PROBLÈME ! 0 est "falsy" pour ||
console.log(score ?? 10); // 0  — CORRECT ! 0 n'est pas null/undefined

const texte = "";

console.log(texte || "défaut"); // "défaut" — "" est "falsy" pour ||
console.log(texte ?? "défaut"); // ""       — "" n'est pas null/undefined

Règle simple : utilise ?? quand 0, "" ou false sont des valeurs valides que tu veux conserver. Utilise || quand tu veux une valeur de remplacement pour tout ce qui est "vide" (0, "", false, null, undefined).

Le combo magique : ?. + ??

Ces deux opérateurs sont faits pour travailler ensemble. L'optional chaining te donne undefined si le chemin n'existe pas, et le nullish coalescing te donne une valeur par défaut : 

const utilisateur = {
    nom: "Jean-Michel",
    préférences: {
        thème: "dark"
    }
};

// Si préférences.langue n'existe pas → "fr"
const langue = utilisateur.préférences?.langue ?? "fr";
console.log(langue); // "fr"

// Si préférences.thème existe → l'utiliser
const thème = utilisateur.préférences?.thème ?? "light";
console.log(thème); // "dark"

// Si l'utilisateur n'a pas de préférences du tout
const user2 = { nom: "Alice" };
const thème2 = user2.préférences?.thème ?? "light";
console.log(thème2); // "light" (pas de crash !)

Sans ces opérateurs, il aurait fallu écrire : user2.préférences && user2.préférences.thème ? user2.préférences.thème : "light". Beaucoup moins lisible.

Assignation logique ??=, ||=, &&=

Pour finir, voici trois raccourcis d'assignation introduits récemment. Ils combinent un opérateur logique avec une assignation : 

let nom = null;
nom ??= "Anonyme"; // Assigne seulement si null/undefined
console.log(nom); // "Anonyme"

let score = 0;
score ??= 100; // 0 n'est pas null/undefined → pas d'assignation
console.log(score); // 0

let debug = true;
debug &&= false; // Assigne seulement si la valeur actuelle est truthy
console.log(debug); // false

Résumé

Opérateur Nom Rôle
? : Ternaire if/else en une expression
?. Optional chaining Accès sécurisé aux propriétés
?? Nullish coalescing Valeur par défaut (null/undefined uniquement)
|| OU logique Valeur par défaut (toutes valeurs falsy)
??= Nullish assignment Assigne si null/undefined

À toi de jouer

  1. Ouvre la console (F12 → Console)
  2. Teste le ternaire : const x = 5 > 3 ? "oui" : "non"
  3. Crée un objet sans certaines propriétés et teste ?.
  4. Compare 0 || 10 et 0 ?? 10 — observe la différence
  5. Combine ?. et ?? sur un objet imbriqué
