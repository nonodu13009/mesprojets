Les Fonctions Fléchées

Leçon 3 · Chapitre 24 · Syntaxe simplifiée et meilleures pratiques

Pourquoi ce chapitre en premier ?

En leçon 2, tu as appris à déclarer des fonctions avec le mot-clé function. Ça fonctionne très bien, mais depuis ES6, il existe une syntaxe plus courte que tu verras partout dans le code moderne : la fonction fléchée (arrow function).

Tous les chapitres suivants de cette leçon l'utilisent. C'est donc la première brique à poser.

De la fonction classique à la fléchée

Partons d'une fonction classique et transformons-la étape par étape : 

// 1. Fonction classique
function doubler(nombre) {
    return nombre * 2;
}

// 2. La même, stockée dans une constante (expression de fonction)
const doubler = function(nombre) {
    return nombre * 2;
};

// 3. On remplace "function" par une flèche => après les paramètres
const doubler = (nombre) => {
    return nombre * 2;
};

// 4. Version courte : une seule expression → pas d'accolades, return implicite
const doubler = (nombre) => nombre * 2;

// 5. Un seul paramètre → les parenthèses deviennent optionnelles
const doubler = nombre => nombre * 2;

console.log(doubler(5)); // 10 — dans tous les cas !

Les cinq versions font exactement la même chose. La fléchée n'ajoute pas de fonctionnalité : elle rend le code plus court et plus lisible.

Les règles de syntaxe

Trois cas à connaître selon le nombre de paramètres : 

// Aucun paramètre : parenthèses vides obligatoires
const direBonjour = () => "Bonjour !";

// Un paramètre : parenthèses optionnelles
const carre = x => x * x;

// Plusieurs paramètres : parenthèses obligatoires
const additionner = (a, b) => a + b;

Et deux cas selon le corps de la fonction : 

// Une seule expression : return implicite (pas d'accolades)
const tripler = x => x * 3;

// Plusieurs instructions : accolades + return explicite
const calculerTTC = (prixHT) => {
    const tva = prixHT * 0.2;
    return prixHT + tva;
};

Piège classique : si tu mets des accolades, le return redevient obligatoire. Sans lui, la fonction renvoie undefined.

Renvoyer un objet : le piège des accolades

Que se passe-t-il si tu veux renvoyer directement un objet { } en version courte ? JavaScript confond les accolades de l'objet avec celles du corps de fonction. La solution : entourer l'objet de parenthèses. 

// ❌ Ne marche pas : JS croit que { } est le corps de la fonction
const creerUtilisateur = nom => { nom: nom };

// ✅ Avec des parenthèses autour de l'objet
const creerUtilisateur = nom => ({ nom: nom });

console.log(creerUtilisateur("Nogaro")); // { nom: "Nogaro" }

Le vrai atout : les callbacks

C'est dans les callbacks (fonctions passées en argument à d'autres fonctions) que la syntaxe fléchée brille vraiment. Compare : 

const nombres = [1, 2, 3, 4, 5];

// Avec des fonctions classiques : verbeux
const doubles = nombres.map(function(n) {
    return n * 2;
});

// Avec des fléchées : une ligne, lisible
const doubles2 = nombres.map(n => n * 2);

console.log(doubles2); // [2, 4, 6, 8, 10]

Tu retrouveras cette écriture dans le chapitre sur les arrays d'objets avec map, filter et find.

Meilleures pratiques

  • Privilégie la fléchée pour les callbacks et les petites fonctions utilitaires : c'est là qu'elle apporte le plus.
  • Reste cohérent : dans un même projet, choisis un style et garde-le.
  • Version courte = une seule expression. Dès que la logique dépasse une ligne, passe aux accolades avec return explicite : la lisibilité prime sur la concision.
  • Attention au this : une fléchée n'a pas son propre this, elle hérite de celui du contexte englobant. C'est pour ça qu'on évite les fléchées comme méthodes d'objet — tu verras ça au chapitre Méthodes des objets.

À toi de jouer

  1. Ouvre la console (F12 → Console)
  2. Écris une fonction classique function saluer(nom) { return "Salut " + nom; }
  3. Réécris-la en fléchée longue : const saluer = (nom) => { return "Salut " + nom; };
  4. Puis en fléchée courte : const saluer = nom => "Salut " + nom;
  5. Crée une fléchée à deux paramètres qui multiplie deux nombres
  6. Teste [1, 2, 3].map(n => n * 10) et observe le résultat
