Promesses et async/await

Leçon 3 · Chapitre 28 · Gérer le code qui prend du temps

Un nouveau type de problème

Jusqu'ici, tout le code que tu as écrit s'exécutait instantanément, ligne après ligne. Mais dans la vraie vie, beaucoup d'opérations prennent du temps : charger des données depuis un serveur, lire un fichier, attendre la réponse d'une API, ou même simplement attendre 3 secondes avant d'afficher un message.

Si JavaScript attendait sagement que chaque opération se termine avant de passer à la suivante, ta page web serait figée pendant chaque chargement. Ce n'est évidemment pas acceptable. C'est pour ça que JavaScript est asynchrone : il lance l'opération et continue d'exécuter le reste du code sans attendre.

Ce chapitre t'apprend à gérer cette asynchronicité proprement, avec les Promesses et la syntaxe async/await.

Synchrone vs asynchrone : la différence en action

Avant de plonger dans les Promesses, observons le comportement asynchrone de JavaScript avec un exemple simple : 

// Code synchrone : chaque ligne attend la précédente
console.log("1");
console.log("2");
console.log("3");
// Affiche : 1, 2, 3 (dans l'ordre, comme prévu)

// Code asynchrone : setTimeout lance un timer, mais n'attend pas
console.log("A");
setTimeout(() => console.log("B"), 1000);
console.log("C");
// Affiche : A, C, B
// "B" arrive 1 seconde après, APRÈS "C" !

C'est surprenant la première fois : "C" s'affiche avant "B" ! JavaScript n'a pas attendu la fin du timer. Il a lancé le setTimeout, puis est immédiatement passé à la ligne suivante. Quand le timer se termine, la fonction callback est exécutée.

Maintenant, imagine que tu charges des données utilisateur depuis un serveur. Tu ne peux pas utiliser les données tant qu'elles ne sont pas arrivées. Comment dire à JavaScript : "quand les données arrivent, alors fais ceci" ? C'est là que les Promesses entrent en jeu.

Les Promesses (Promise)

Une Promise est un objet qui représente une valeur qui n'est pas encore disponible mais qui le sera dans le futur. C'est comme un ticket de commande au restaurant : tu ne l'as pas encore, mais tu as la promesse que ça va arriver.

Une Promise a 3 états possibles :

État Signification
pending En attente — l'opération est en cours
fulfilled Tenue — l'opération a réussi, on a le résultat
rejected Rejetée — une erreur s'est produite 
// Créer une promesse
const maPromesse = new Promise((resolve, reject) => {
    const succès = true;

    if (succès) {
        resolve("Tout s'est bien passé !"); // → fulfilled
    } else {
        reject("Quelque chose a échoué.");  // → rejected
    }
});

// Consommer la promesse
maPromesse
    .then((résultat) => console.log(résultat))  // si fulfilled
    .catch((erreur) => console.log(erreur));     // si rejected

.then() est appelé quand la promesse est tenue (succès), .catch() quand elle est rejetée (erreur). C'est un peu comme un if/else pour le code asynchrone.

Exemple concret : simuler un appel serveur

En vrai, les Promesses sont rarement créées à la main. Mais pour bien comprendre le mécanisme, simulons un appel API qui met 1,5 seconde à répondre : 

function chargerUtilisateur(id) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            if (id > 0) {
                resolve({ id: id, nom: "Jean-Michel", ville: "Marseille" });
            } else {
                reject("ID invalide");
            }
        }, 1500); // Simule 1,5s de chargement
    });
}

// Utilisation
chargerUtilisateur(1)
    .then((user) => console.log(`Bonjour ${user.nom} !`))
    .catch((err) => console.log(`Erreur : ${err}`));

console.log("Ce message s'affiche AVANT le résultat !");

// Affiche immédiatement : "Ce message s'affiche AVANT le résultat !"
// Puis après 1,5s : "Bonjour Jean-Michel !"

async/await : écrire de l'asynchrone comme du synchrone

La syntaxe .then().catch() fonctionne, mais quand tu enchaînes plusieurs opérations asynchrones, ça devient vite difficile à lire. async/await est du sucre syntaxique : ça fait exactement la même chose, mais avec une syntaxe qui ressemble à du code synchrone classique. 

// Le mot-clé "async" devant la fonction
async function afficherUtilisateur() {
    try {
        // "await" met en pause CETTE fonction jusqu'au résultat
        const user = await chargerUtilisateur(1);
        console.log(`Bonjour ${user.nom} de ${user.ville} !`);
    } catch (erreur) {
        console.log(`Erreur : ${erreur}`);
    }
}

afficherUtilisateur();
// Après 1,5s : "Bonjour Jean-Michel de Marseille !"

Compare avec la version .then() précédente : c'est beaucoup plus lisible. On lit le code de haut en bas, comme du code normal. Les erreurs sont gérées avec un try/catch classique au lieu d'un .catch() chaîné.

Deux règles à retenir : await ne peut être utilisé que dans une fonction déclarée async. Et une fonction async retourne toujours une Promesse, même si tu ne le vois pas.

fetch : faire de vrais appels HTTP

Assez de simulation ! fetch est la méthode native du navigateur pour faire des requêtes HTTP (appeler une API, charger des données JSON...). Elle retourne une Promesse, donc tout ce que tu viens d'apprendre s'applique : 

// Version avec .then() (pour comprendre)
fetch("https://jsonplaceholder.typicode.com/users/1")
    .then((response) => response.json()) // convertir la réponse en JSON
    .then((data) => console.log(data.name))
    .catch((err) => console.log("Erreur :", err));

// Version avec async/await (à privilégier)
async function chargerDepuisAPI() {
    try {
        const response = await fetch("https://jsonplaceholder.typicode.com/users/1");
        const data = await response.json();
        console.log(data.name); // "Leanne Graham"
    } catch (erreur) {
        console.log("Erreur :", erreur);
    }
}

chargerDepuisAPI();

Note qu'il y a deux await : un pour la requête HTTP elle-même, et un pour la conversion de la réponse en JSON. Ce sont deux opérations asynchrones distinctes.

Quand utiliser quoi ?

Syntaxe Avantage Quand l'utiliser
.then().catch() Chaînage facile, compact Opérations simples, callbacks rapides
async/await Lisibilité, gestion d'erreurs claire Logique complexe, plusieurs appels successifs

En pratique, async/await est devenu le standard dans le code moderne. Tu verras .then() surtout dans du code existant ou pour des cas simples en une ligne.

À toi de jouer

  2. Teste le comportement asynchrone : console.log("A"); setTimeout(() => console.log("B"), 2000); console.log("C");
  3. Colle la fonction chargerUtilisateur et appelle-la avec .then()
  4. Appelle la même fonction avec async/await dans une fonction async
  5. Teste un vrai fetch avec l'URL jsonplaceholder ci-dessus
  6. Provoque une erreur (mauvaise URL) et observe le catch
