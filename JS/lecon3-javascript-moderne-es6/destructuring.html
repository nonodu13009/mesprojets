Destructuration

Leçon 3 · Chapitre 24 · Extraire des valeurs en une ligne

Pourquoi ce chapitre en premier ?

En leçon 2, tu as appris à créer des fonctions et à manipuler des objets et des tableaux. Tu as sûrement écrit du code comme const nom = utilisateur.nom plusieurs fois de suite pour extraire des valeurs.

La destructuration est la première syntaxe ES6 que tu vas apprendre parce qu'elle résout un problème que tu as déjà rencontré : la répétition inutile. Et surtout, tu la retrouveras partout dans la suite de cette leçon — dans les paramètres de fonctions, avec le spread/rest, dans les méthodes de tableaux, etc.

Commence par bien la comprendre, et tout le reste coulera naturellement.

Le problème : du code répétitif

Imagine que tu récupères un objet depuis une API (par exemple les infos d'un utilisateur). Pour utiliser ses propriétés, tu dois les extraire une par une : 

const utilisateur = {
    nom: "Nogaro",
    prenom: "Jean-Michel",
    age: 30
};

// Sans destructuration : on répète "utilisateur." trois fois
const nom = utilisateur.nom;
const prenom = utilisateur.prenom;
const age = utilisateur.age;

Ça fonctionne, mais c'est verbeux. Avec 10 propriétés, tu aurais 10 lignes quasi identiques. ES6 a introduit une syntaxe pour faire ça en une seule ligne.

La solution : destructuration d'objet

La destructuration permet d'extraire des propriétés d'un objet directement dans des variables. On utilise des accolades { } à gauche du signe = : 

const utilisateur = {
    nom: "Nogaro",
    prenom: "Jean-Michel",
    age: 30
};

// Avec destructuration : une seule ligne
const { nom, prenom, age } = utilisateur;

console.log(nom);    // "Nogaro"
console.log(prenom); // "Jean-Michel"
console.log(age);    // 30

C'est exactement le même résultat qu'avant, mais en une ligne au lieu de trois. JavaScript regarde les noms entre les accolades et va chercher les propriétés correspondantes dans l'objet.

Règle importante : les noms des variables doivent correspondre exactement aux noms des propriétés de l'objet. Si tu écris { name } au lieu de { nom }, tu obtiendras undefined.

Renommer au passage

Parfois, le nom de la propriété ne te convient pas — il est trop court, ou il entre en conflit avec une variable existante. Tu peux renommer avec la syntaxe propriété: nouveauNom : 

const { nom: nomFamille, prenom: prenomUtilisateur } = utilisateur;

console.log(nomFamille);        // "Nogaro"
console.log(prenomUtilisateur); // "Jean-Michel"

Lis ça comme : "prends la propriété nom et mets-la dans une variable appelée nomFamille".

Valeurs par défaut

Que se passe-t-il si tu essaies de destructurer une propriété qui n'existe pas dans l'objet ? Tu obtiens undefined. Pour éviter ça, tu peux définir une valeur par défaut : 

const { nom, prenom, ville = "Marseille" } = utilisateur;

console.log(ville); // "Marseille" (la propriété n'existe pas → valeur par défaut)

Si l'objet avait eu une propriété ville, c'est sa valeur qui aurait été utilisée. La valeur par défaut n'est qu'un filet de sécurité.

Et pour les tableaux ?

La destructuration fonctionne aussi avec les tableaux. La différence : on utilise des crochets [ ] au lieu des accolades, et c'est l'ordre qui compte (pas les noms) : 

const couleurs = ["rouge", "vert", "bleu"];

const [première, deuxième, troisième] = couleurs;

console.log(première);  // "rouge"
console.log(deuxième);  // "vert"
console.log(troisième); // "bleu"

// Tu peux ignorer des éléments avec des virgules vides
const [, , dernière] = couleurs;
console.log(dernière); // "bleu"

Tu verras cette syntaxe très souvent avec React (le fameux const [count, setCount] = useState(0)), mais pas besoin de connaître React pour comprendre le mécanisme.

Application concrète : dans les paramètres de fonction

C'est là où la destructuration devient vraiment puissante. Au lieu de recevoir un objet entier dans une fonction et d'accéder à ses propriétés une par une, tu peux destructurer directement dans les paramètres : 

// Avant : on reçoit l'objet entier
function afficherUtilisateur(utilisateur) {
    console.log(utilisateur.prenom + " " + utilisateur.nom);
}

// Après : on destructure dans les paramètres
function afficherUtilisateur({ prenom, nom }) {
    console.log(`${prenom} ${nom}`);
}

afficherUtilisateur({ nom: "Nogaro", prenom: "Jean-Michel", age: 30 });
// "Jean-Michel Nogaro"

L'avantage : en lisant juste la signature de la fonction ({ prenom, nom }), tu sais immédiatement quelles propriétés elle utilise, sans avoir à lire tout le corps de la fonction.

Pour aller plus loin : destructuration imbriquée

Quand un objet contient d'autres objets, tu peux aller chercher des valeurs en profondeur : 

const entreprise = {
    nom: "Allianz",
    adresse: {
        ville: "Marseille",
        codePostal: "13008"
    }
};

const { adresse: { ville, codePostal } } = entreprise;

console.log(ville);      // "Marseille"
console.log(codePostal); // "13008"

Attention : dans cet exemple, la variable adresse n'est pas créée. Seules ville et codePostal le sont. Si tu veux aussi adresse, il faut la destructurer séparément.

À toi de jouer

  1. Ouvre la console (F12 → Console)
  2. Crée un objet const personne = { nom: "Dupont", age: 25, ville: "Paris" }
  3. Destructure-le : const { nom, age } = personne;
  4. Teste le renommage : const { nom: n } = personne;
  5. Crée un tableau et destructure-le avec des crochets
  6. Crée une fonction qui prend un objet destructuré en paramètre

La suite

Tu sais maintenant extraire des valeurs depuis des objets et des tableaux. Mais que se passe-t-il quand tu veux faire l'inverse : étaler un tableau dans un autre, ou rassembler des arguments ? C'est exactement ce que font les opérateurs Spread et Rest, qu'on va voir dans le prochain chapitre.
