Déstructuration

Leçon 3 · Chapitre 26 · Simplifie et optimise ton code

Le problème : du code répétitif

Au chapitre précédent, tu as appris à lire les propriétés d'un objet. Pour utiliser plusieurs propriétés, tu dois les extraire une par une : 

const utilisateur = {
    nom: "Nogaro",
    prenom: "Jean-Michel",
    age: 30
};

// Sans destructuration : on répète "utilisateur." trois fois
const nom = utilisateur.nom;
const prenom = utilisateur.prenom;
const age = utilisateur.age;

Ça fonctionne, mais c'est verbeux. Avec 10 propriétés, tu aurais 10 lignes quasi identiques. ES6 a introduit une syntaxe pour faire ça en une seule ligne.

La solution : destructuration d'objet

La destructuration permet d'extraire des propriétés d'un objet directement dans des variables. On utilise des accolades { } à gauche du signe = : 

// Avec destructuration : une seule ligne
const { nom, prenom, age } = utilisateur;

console.log(nom);    // "Nogaro"
console.log(prenom); // "Jean-Michel"
console.log(age);    // 30

C'est exactement le même résultat qu'avant, mais en une ligne au lieu de trois. JavaScript regarde les noms entre les accolades et va chercher les propriétés correspondantes dans l'objet.

Règle importante : les noms des variables doivent correspondre exactement aux noms des propriétés de l'objet. Si tu écris { name } au lieu de { nom }, tu obtiendras undefined.

Renommer et valeurs par défaut

Deux options bien pratiques : renommer une propriété au passage avec propriété: nouveauNom, et prévoir une valeur par défaut avec = si la propriété n'existe pas : 

// Renommer : "prends nom, mets-le dans nomFamille"
const { nom: nomFamille } = utilisateur;
console.log(nomFamille); // "Nogaro"

// Valeur par défaut si la propriété est absente
const { ville = "Marseille" } = utilisateur;
console.log(ville); // "Marseille" (pas de propriété ville → défaut)

Et pour les tableaux ?

La destructuration fonctionne aussi avec les tableaux. La différence : on utilise des crochets [ ] au lieu des accolades, et c'est l'ordre qui compte (pas les noms) : 

const couleurs = ["rouge", "vert", "bleu"];

const [première, deuxième, troisième] = couleurs;

console.log(première);  // "rouge"
console.log(deuxième);  // "vert"

// Tu peux ignorer des éléments avec des virgules vides
const [, , dernière] = couleurs;
console.log(dernière); // "bleu"

Tu verras cette syntaxe très souvent avec React (le fameux const [count, setCount] = useState(0)), mais pas besoin de connaître React pour comprendre le mécanisme.

Application concrète : dans les paramètres de fonction

C'est là où la destructuration devient vraiment puissante. Au lieu de recevoir un objet entier dans une fonction, tu peux destructurer directement dans les paramètres : 

// Avant : on reçoit l'objet entier
const afficher = (utilisateur) => {
    console.log(utilisateur.prenom + " " + utilisateur.nom);
};

// Après : on destructure dans les paramètres
const afficher2 = ({ prenom, nom }) => {
    console.log(`${prenom} ${nom}`);
};

afficher2({ nom: "Nogaro", prenom: "Jean-Michel", age: 30 });
// "Jean-Michel Nogaro"

L'avantage : en lisant juste la signature ({ prenom, nom }), tu sais immédiatement quelles propriétés la fonction utilise, sans avoir à lire tout son corps.

À toi de jouer

  1. Ouvre la console (F12 → Console)
  2. Crée un objet const personne = { nom: "Dupont", age: 25, ville: "Paris" }
  3. Destructure-le : const { nom, age } = personne;
  4. Teste le renommage : const { nom: n } = personne;
  5. Crée un tableau et destructure-le avec des crochets
  6. Crée une fonction fléchée qui prend un objet destructuré en paramètre
