Application Météo — Partie 2

Leçon 6 · Projet 37 · L'interface complète

Le brief

Transformer le prototype de la partie 1 en application aboutie : carte météo design, icône correspondant au ciel, détails complets (ressenti, humidité, vent), état de chargement et petites attentions qui font la différence. C'est le plus gros chapitre du cours (1 h 38) — prends ton temps.

Ce que tu vas pratiquer

  • Générer du HTML depuis des données : createElement ou template literals (leçon 4)
  • classList pour les états visuels : chargement, erreur, succès (leçon 4)
  • Destructuration d'objets imbriqués pour extraire les données API (leçon 3)
  • CSS : carte centrée, ombres douces, transitions

Fonctionnalités attendues

  • Carte météo : nom de la ville + pays, grande température, description
  • Icône météo (l'API fournit un code icône : https://openweathermap.org/img/wn/CODE@2x.png)
  • Trois détails : ressenti (feels_like), humidité, vitesse du vent
  • Indicateur de chargement pendant l'appel API
  • Message d'erreur stylé (ville inconnue, réseau coupé)
  • Recherche au clavier : la touche Entrée déclenche la recherche

Plan de construction

  1. Maquette statique d'abord : construis la carte en HTML/CSS avec des données en dur ("Marseille, 24°, ensoleillé"). Peaufine le design sans te soucier du JS
  2. Fonction afficherMeteo(donnees) : remplace les données en dur par celles de l'objet API. Destructure ce dont tu as besoin dès l'entrée de la fonction
  3. Icône : récupère weather[0].icon et compose l'URL de l'image dans un <img>
  4. États d'interface : trois classes CSS — .chargement, .erreur, .succes — et une fonction qui bascule de l'une à l'autre avec classList
  5. Touche Entrée : écoute l'événement keydown sur l'input, teste event.key === "Enter"
  6. Finitions : vider l'input après recherche, arrondir les températures avec Math.round, première lettre de la description en majuscule

Les pièges classiques

  • Afficher avant d'avoir reçu : tout affichage doit se faire après le await, jamais avant
  • Résultats qui s'empilent : vide la zone de résultat (innerHTML = "" ou replaceChildren()) avant chaque nouvelle recherche
  • innerHTML + saisie utilisateur : le nom de ville vient de l'utilisateur — préfère textContent pour l'afficher (réflexe anti-XSS)
  • Spinner fantôme : pense à masquer le chargement aussi en cas d'erreur (c'est le rôle de finally)

Pour aller plus loin

  • Mémorise la dernière ville dans localStorage et recharge-la à l'ouverture
  • Change le fond de la carte selon la météo (bleu = pluie, jaune = soleil...)
  • Ajoute la géolocalisation avec navigator.geolocation
  • Affiche les prévisions sur 5 jours (endpoint /forecast de l'API)
