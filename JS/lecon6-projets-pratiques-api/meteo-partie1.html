Application Météo — Partie 1

Leçon 6 · Projet 36 · Structure, API et premières données

Le brief

Construire une application qui affiche la météo actuelle d'une ville saisie par l'utilisateur : température, description du ciel, humidité, vent. Cette partie 1 pose les fondations : la structure HTML, l'appel à l'API et l'affichage des données brutes.

C'est LE projet type du développeur web : un formulaire, un appel d'API, un affichage dynamique. Si tu sais faire ça, tu sais faire 80 % des applications du web.

Ce que tu vas pratiquer

  • fetch + async/await + try/catch (leçon 5)
  • Lire un objet JSON complexe renvoyé par une API (leçon 3)
  • querySelector, textContent, addEventListener (leçon 4)
  • Template literals pour composer l'URL et les textes

L'API météo

Le cours utilise une API météo (type OpenWeatherMap). Le principe est toujours le même :

  1. Crée un compte gratuit sur le site de l'API pour obtenir une clé API
  2. Compose l'URL avec la ville et ta clé : https://api.openweathermap.org/data/2.5/weather?q=Marseille&appid=TA_CLE&units=metric&lang=fr
  3. Teste l'URL directement dans le navigateur pour voir le JSON brut
  4. Repère les données utiles : main.temp, weather[0].description, main.humidity, wind.speed

Sécurité : une clé API dans du JavaScript front est visible par tout le monde. Acceptable pour un projet d'apprentissage avec une clé gratuite, mais ne fais jamais ça avec une clé payante ou sensible.

Plan de construction

  1. Structure HTML : un <input> pour la ville, un <button> "Rechercher", une zone <div id="resultat"> vide
  2. Brancher l'événement : au clic sur le bouton, récupère la valeur de l'input et affiche-la en console.log — vérifie que ça marche avant d'aller plus loin
  3. Fonction obtenirMeteo(ville) : async, compose l'URL, fetch, teste reponse.ok, renvoie le JSON
  4. Afficher en console : appelle la fonction et explore l'objet reçu — repère où se cachent la température et la description
  5. Afficher dans la page : injecte ville, température arrondie et description dans #resultat avec textContent ou createElement
  6. Gérer les erreurs : ville inexistante → message clair pour l'utilisateur, pas un écran vide

Les pièges classiques

  • Température en Kelvin : sans units=metric dans l'URL, tu afficheras 293° à Marseille
  • weather est un tableau : c'est donnees.weather[0].description, pas donnees.weather.description
  • 404 silencieux : fetch ne lève pas d'erreur pour une ville inconnue — pense au if (!reponse.ok)
  • Input vide : vérifie que l'utilisateur a saisi quelque chose avant d'appeler l'API

Checklist de fin de partie 1

  • Je saisis "Marseille", je clique, la température s'affiche
  • Je saisis "Zzzville", un message d'erreur clair s'affiche
  • Je saisis rien, l'application ne plante pas
  • Le code utilise async/await avec try/catch

La suite

Les données arrivent et s'affichent : les fondations sont posées. La partie 2 transforme ce prototype en vraie application : interface soignée, icônes météo, détails complets et confort d'usage.
