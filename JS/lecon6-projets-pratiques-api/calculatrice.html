Calculatrice Moderne

Leçon 6 · Projet 39 · Gestion d'état et délégation d'événements

Le brief

Construire une calculatrice complète façon iOS/Android : écran d'affichage, grille de touches (chiffres, opérateurs, égal, effacer), et les quatre opérations. Le défi n'est pas visuel, il est logique : gérer l'état de la calculatrice sans s'emmêler.

Ce que tu vas pratiquer

  • Gestion d'état : un objet qui mémorise où en est le calcul (leçon 3, objets et méthodes)
  • Délégation d'événements : UN écouteur sur la grille plutôt que 20 sur les touches
  • CSS Grid pour la grille de touches
  • switch pour dispatcher les opérateurs (leçon 2)

Fonctionnalités attendues

  • Écran affichant la saisie en cours et le résultat
  • Touches 0-9, virgule, + − × ÷, =, C (tout effacer)
  • Enchaînement des calculs : 2 + 3 × 4 touche par touche
  • Division par zéro gérée proprement (message, pas Infinity)
  • Design soigné : grille régulière, touches opérateurs colorées, effet au survol

Le cœur du projet : l'état

Une calculatrice doit se souvenir de trois choses. Modélise-les dans un objet dès le départ : 

const etat = {
    premierNombre: null,   // le nombre avant l'opérateur
    operateur: null,       // "+", "-", "×", "÷"
    saisieCourante: "0"    // ce qui s'affiche à l'écran (une CHAÎNE)
};

Chaque touche ne fait qu'une chose : mettre à jour cet état, puis rafraîchir l'écran. Si tu tiens cette discipline, la calculatrice reste simple.

Plan de construction

  1. HTML/CSS : l'écran + la grille de touches en CSS Grid (grid-template-columns: repeat(4, 1fr))
  2. Délégation : un seul addEventListener("click") sur la grille ; identifie la touche cliquée avec event.target et un attribut data-touche
  3. Saisie des chiffres : concatène à saisieCourante (c'est une chaîne : "1" + "2" = "12" — c'est voulu !) et rafraîchis l'écran
  4. Touche opérateur : stocke saisieCourante dans premierNombre, mémorise l'opérateur, réinitialise la saisie
  5. Touche égal : convertis les deux nombres avec Number(), calcule selon l'opérateur (un switch), affiche le résultat
  6. Touche C : remets l'état à ses valeurs initiales
  7. Cas limites : double virgule interdite, division par zéro, appui sur = sans opérateur

Les pièges classiques

  • Le piège n°1 du projet : oublier Number() et faire "2" + "3" = "23". Chaîne pour l'affichage, nombre pour le calcul
  • 0,1 + 0,2 = 0,30000000000000004 : les flottants ! Arrondis l'affichage avec parseFloat(resultat.toFixed(10))
  • Ne JAMAIS utiliser eval() pour calculer : faille de sécurité majeure. Ton switch fait le travail proprement
  • Enchaîner après = : décide quoi faire si l'utilisateur tape un chiffre juste après un résultat (nouvelle saisie ? continuation ?)

Pour aller plus loin

  • Support du clavier physique (événement keydown)
  • Touche ± (inverser le signe) et %
  • Historique des calculs affiché au-dessus de l'écran
  • Calculs en chaîne : 2 + 3 + 4 sans appuyer sur = entre chaque
