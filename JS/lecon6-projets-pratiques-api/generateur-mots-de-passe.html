Générateur de Mots de Passe

Leçon 6 · Projet 38 · Logique pure et manipulation de chaînes

Le brief

Construire un outil qui génère des mots de passe aléatoires robustes : l'utilisateur choisit la longueur et les types de caractères (majuscules, minuscules, chiffres, symboles), clique, et peut copier le résultat en un clic.

Pas d'API ici : c'est un projet de logique pure. Parfait pour muscler les fonctions, les chaînes et les conditions.

Ce que tu vas pratiquer

  • Math.random() et l'accès aux caractères d'une chaîne par index
  • Les checkboxes : lire l'état checked d'un input (leçon 4)
  • Composer une fonction pure : options en entrée → mot de passe en sortie
  • L'API presse-papiers : navigator.clipboard.writeText() (asynchrone, leçon 5 !)

Fonctionnalités attendues

  • Slider ou input numérique pour la longueur (8 à 32 caractères)
  • 4 checkboxes : majuscules, minuscules, chiffres, symboles
  • Bouton "Générer" → le mot de passe s'affiche
  • Bouton "Copier" → mot de passe dans le presse-papiers + feedback visuel ("Copié !")
  • Au moins une case doit être cochée, sinon message d'erreur

Plan de construction

  1. HTML : le formulaire d'options + zone de résultat + les deux boutons
  2. Les alphabets : quatre constantes — const MINUSCULES = "abcdefghijklmnopqrstuvwxyz", pareil pour majuscules, chiffres et symboles
  3. Caractère aléatoire : écris const caractereAleatoire = chaine => chaine[Math.floor(Math.random() * chaine.length)] et teste-la en console
  4. Construire l'alphabet actif : selon les cases cochées, concatène les alphabets dans une seule chaîne
  5. Boucle de génération : une boucle for de la longueur choisie, qui ajoute un caractère aléatoire à chaque tour
  6. Affichage + copie : injecte le résultat, puis branche navigator.clipboard.writeText(motDePasse) sur le bouton copier (avec await)

Les pièges classiques

  • Valeur d'input = chaîne : la longueur lue vaut "12", pas 12 — convertis avec Number()
  • Aucune case cochée : alphabet vide → boucle qui produit undefined. Valide avant de générer
  • clipboard est asynchrone : c'est une promesse, gère-la avec await et try/catch
  • Feedback copie : pense à remettre le texte du bouton à l'état initial après 2 secondes (setTimeout !)

Pour aller plus loin

  • Indicateur de force du mot de passe (faible / moyen / fort) selon longueur et variété
  • Garantir au moins un caractère de chaque type coché
  • Historique des 5 derniers mots de passe générés
  • Pour du vrai usage sécurité : remplace Math.random() par crypto.getRandomValues(), l'aléatoire cryptographique du navigateur
