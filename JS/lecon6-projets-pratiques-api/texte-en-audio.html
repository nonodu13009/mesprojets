Convertisseur Texte-en-Audio

Leçon 6 · Projet 41 · L'API Web Speech du navigateur

Le brief

Construire une application qui lit à voix haute n'importe quel texte saisi : choix de la voix, réglage de la vitesse et du ton, boutons lecture / pause / stop. Le tout sans aucun serveur : le navigateur embarque nativement un moteur de synthèse vocale, la Web Speech API.

Ce que tu vas pratiquer

  • Découvrir une API navigateur par sa documentation (MDN) — compétence clé de développeur
  • Objets et propriétés : configurer un SpeechSynthesisUtterance (leçon 3)
  • Remplir un <select> dynamiquement à partir d'un tableau d'objets (leçons 3 et 4)
  • Inputs range (sliders) et leur événement input

L'API Web Speech en 30 secondes

Teste ça immédiatement dans la console — ta machine va parler : 

// L'objet "énoncé" : le texte + sa configuration
const enonce = new SpeechSynthesisUtterance("Bonjour Jean-Michel !");
enonce.lang = "fr-FR";
enonce.rate = 1;   // vitesse (0.1 à 10)
enonce.pitch = 1;  // ton (0 à 2)

// Le moteur global du navigateur
speechSynthesis.speak(enonce);

// Les voix disponibles (un array d'objets !)
console.log(speechSynthesis.getVoices());

Fonctionnalités attendues

  • Zone de texte multiligne (<textarea>)
  • Menu déroulant des voix disponibles (filtrées en français, ou toutes)
  • Deux sliders : vitesse et ton, avec leur valeur affichée en direct
  • Boutons Lire / Pause / Reprendre / Stop
  • Indicateur visuel pendant la lecture (bouton actif, animation...)

Plan de construction

  1. Console d'abord : joue avec speechSynthesis dans la console jusqu'à bien comprendre speak, pause, resume, cancel
  2. HTML/CSS : textarea, select, deux sliders, la barre de boutons
  3. Charger les voix : getVoices() renvoie un array d'objets — forEach pour créer les <option> du select
  4. Lecture : au clic sur Lire, crée l'énoncé, applique voix / vitesse / ton depuis les contrôles, lance speak
  5. Contrôles : branche pause(), resume() et cancel() sur les autres boutons
  6. État visuel : les événements start et end de l'énoncé te disent quand activer/désactiver l'indicateur de lecture

Les pièges classiques

  • Liste de voix vide : getVoices() se charge de façon asynchrone. Écoute l'événement speechSynthesis.onvoiceschanged pour remplir le select au bon moment
  • Lectures qui s'empilent : appelle speechSynthesis.cancel() avant chaque nouveau speak
  • Slider = chaîne : comme toujours, Number() avant d'affecter à rate ou pitch
  • Voix selon l'OS : chaque machine a des voix différentes — ton select doit s'adapter, pas coder les noms en dur

Pour aller plus loin

  • Surligner le mot en cours de lecture (événement boundary de l'énoncé)
  • Mémoriser les réglages préférés dans localStorage
  • Compteur de caractères sous le textarea
  • Explore l'autre moitié de la Web Speech API : la reconnaissance vocale (SpeechRecognition)
