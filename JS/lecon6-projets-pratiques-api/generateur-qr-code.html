Générateur de QR Code

Leçon 6 · Projet 40 · API d'images et expérience utilisateur

Le brief

Construire un outil qui transforme n'importe quel texte ou URL en QR Code : l'utilisateur saisit, le QR Code apparaît, et il peut le télécharger en image. Projet court mais très gratifiant — et l'occasion de découvrir les API qui renvoient des images plutôt que du JSON.

Ce que tu vas pratiquer

  • API par URL : composer une URL de génération avec des paramètres
  • encodeURIComponent() : encoder proprement la saisie utilisateur dans une URL
  • Manipuler des <img> dynamiquement : src, événement load (leçon 4)
  • États d'interface : vide → chargement → affiché (classes CSS)

L'API QR Code

Le cours utilise une API de génération (type api.qrserver.com). Pas de clé nécessaire : toute la configuration passe par l'URL : 

// Structure de l'URL de génération
const texte = encodeURIComponent(saisie);
const url = `https://api.qrserver.com/v1/create-qr-code/?size=200x200&data=${texte}`;

// Il suffit de la mettre dans le src d'une image !
image.src = url;

À comprendre : pas de fetch ici — le navigateur charge l'image lui-même quand tu changes le src. C'est l'événement load de l'image qui te dit quand c'est prêt.

Fonctionnalités attendues

  • Champ texte + bouton "Générer"
  • Choix de la taille (petit / moyen / grand)
  • Affichage du QR Code avec une petite animation d'apparition
  • Bouton "Télécharger" l'image
  • Saisie vide → message d'erreur, pas de QR Code cassé

Plan de construction

  1. HTML/CSS : carte centrée avec input, select de taille, bouton et zone d'image (cachée au départ)
  2. Génération : au clic, valide la saisie, compose l'URL avec encodeURIComponent, affecte-la au src
  3. État de chargement : affiche un message "Génération...", et masque-le dans l'écouteur image.addEventListener("load", ...)
  4. Animation : une classe CSS .visible avec transition d'opacité, ajoutée au load
  5. Téléchargement : fetch l'URL de l'image, convertis en blob (await reponse.blob()), crée un lien avec URL.createObjectURL(blob) et l'attribut download, puis déclenche son click()

Les pièges classiques

  • Caractères spéciaux : sans encodeURIComponent, un texte avec &, ? ou des espaces casse l'URL
  • Ancien QR affiché : pendant le chargement du nouveau, l'ancien reste visible — masque-le d'abord
  • Le téléchargement direct par lien simple ne marche pas : l'attribut download est ignoré pour les images d'un autre domaine, d'où le passage par le blob
  • Événement load jamais déclenché si l'URL est invalide : écoute aussi error sur l'image

Pour aller plus loin

  • Génération en direct pendant la frappe (avec un petit délai — cherche "debounce")
  • Choix des couleurs du QR Code (paramètres color et bgcolor de l'API)
  • Historique des derniers QR Codes générés dans localStorage
  • Bouton "Copier l'image" dans le presse-papiers
